# C1 학습 데이터 관리 및 편향 관리

## 통제 목표

학습 데이터는 출처, 보안, 품질 및 공정성을 보존하는 방식으로 수집, 처리 및 관리되어야 합니다. 이렇게 함으로써 법적 의무를 이행하고 AI 생명주기 전반에 걸쳐 편향, 오염 또는 개인정보 침해의 위험을 줄일 수 있습니다.

---

모든 데이터셋의 검증 가능한 목록을 유지하고, 신뢰할 수 있는 출처만 허용하며, 모든 변경 사항을 감사 가능하도록 기록하십시오.

|   #   | Description                                                                                           | Level | Role |
| :---: | ----------------------------------------------------------------------------------------------------- | :---: | :--: |
| 1.1.1 | 모든 학습 데이터 소스(출처, 관리/소유자, 라이선스, 수집 방법, 사용 목적 제한, 처리 이력)에 대한 최신 재고 목록이 유지되는지 확인하십시오.                    |   1   | D/V  |
| 1.1.2 | 품질, 대표성, 윤리적 출처 및 라이선스 준수가 검증된 데이터 세트만 허용되어야 하며, 이를 통해 오염, 내재된 편향 및 지적 재산권 침해의 위험을 줄여야 합니다.           |   1   | D/V  |
| 1.1.3 | 불필요하거나 민감한 속성이 제외되도록 데이터 최소화가 시행되는지 확인하십시오.                                                           |   1   | D/V  |
| 1.1.4 | 모든 데이터셋 변경 사항이 기록된 승인 워크플로우의 대상인지 확인하십시오.                                                             |   2   | D/V  |
| 1.1.5 | 검증자 상호 검토 또는 합의를 통해 라벨링/주석 품질이 보장되는지 확인하십시오.                                                          |   2   | D/V  |
| 1.1.6 | 주요 학습 데이터셋에 대해 특성, 동기, 구성, 수집 과정, 전처리 및 권장/비권장 사용법을 자세히 설명하는 "데이터 카드" 또는 "데이터셋용 데이터시트"가 유지되는지 확인하십시오. |   2   | D/V  |

접근을 제한하고, 저장 시와 전송 중에 암호화하며, 변조나 도난을 차단하기 위해 무결성을 검증하십시오.

|   #   | Description                                                                                                                  | Level | Role |
| :---: | ---------------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 1.2.1 | 접근 제어가 저장소와 파이프라인을 보호하는지 확인하십시오.                                                                                             |   1   | D/V  |
| 1.2.2 | 데이터셋이 전송 중에 암호화되어 있고, 모든 민감하거나 개인 식별 정보(PII)는 저장 중에도 업계 표준 암호화 알고리즘과 키 관리 방식을 사용하여 암호화되어 있는지 확인하십시오.                         |   2   | D/V  |
| 1.2.3 | 암호화 해시 또는 디지털 서명이 저장 및 전송 중 데이터 무결성을 보장하기 위해 사용되는지 확인하고, 자동화된 이상 탐지 기법이 무단 변경 또는 손상, 특히 표적화된 데이터 중독 시도를 방지하기 위해 적용되는지 검증합니다. |   2   | D/V  |
| 1.2.4 | 데이터셋 버전이 추적되어 롤백이 가능하도록 확인하십시오.                                                                                              |   3   | D/V  |
| 1.2.5 | 데이터 보존 정책, 규제 요구사항을 준수하고 공격 표면을 축소하기 위해 오래된 데이터가 안전하게 삭제되거나 익명화되었는지 확인하십시오.                                                  |   2   | D/V  |

## C1.10 데이터 계보 및 추적성

감사 및 사고 대응을 위해 각 데이터 포인트가 소스에서 모델 입력까지의 전체 경로를 추적하십시오.

|   #    | Description                                                 | Level | Role |
| :----: | ----------------------------------------------------------- | :---: | :--: |
| 1.10.1 | 각 데이터 포인트의 계보가 모든 변환, 증강 및 병합을 포함하여 기록되고 재구성될 수 있는지 확인하십시오. |   2   | D/V  |
| 1.10.2 | 계보 기록이 변경 불가능하며, 안전하게 저장되고, 감사나 조사 시 접근 가능함을 확인하십시오.        |   2   | D/V  |
| 1.10.3 | 계보 추적이 원시 데이터와 합성 데이터 모두를 포함하는지 확인하십시오.                     |   2   | D/V  |

## C1.11 합성 데이터 관리

합성 데이터가 적절하게 관리되고, 라벨링되며, 위험 평가가 이루어지도록 하십시오.

|   #    | Description                                                       | Level | Role |
| :----: | ----------------------------------------------------------------- | :---: | :--: |
| 1.11.1 | 모든 합성 데이터가 파이프라인 전반에서 명확하게 라벨링되고 실제 데이터와 구별 가능하도록 확인하십시오.         |   2   | D/V  |
| 1.11.2 | 합성 데이터의 생성 과정, 매개변수 및 의도된 사용이 문서화되었는지 확인하십시오.                     |   2   | D/V  |
| 1.11.3 | 합성 데이터가 훈련에 사용되기 전에 편향, 개인정보 유출 및 표현 문제에 대해 위험 평가가 이루어졌는지 확인하십시오. |   2   | D/V  |

## C1.12 데이터 접근 모니터링 및 이상 탐지

내부자 위협이나 데이터 유출을 탐지하기 위해 학습 데이터에 대한 비정상적인 접근을 모니터링하고 경고합니다.

|   #    | Description                                                           | Level | Role |
| :----: | --------------------------------------------------------------------- | :---: | :--: |
| 1.12.1 | 모든 학습 데이터 접근이 사용자, 시간 및 작업을 포함하여 기록되는지 확인하십시오.                        |   2   | D/V  |
| 1.12.2 | 액세스 로그가 정기적으로 검토되어 대규모 내보내기 또는 새로운 위치에서의 접근과 같은 비정상적인 패턴이 있는지 확인하십시오. |   2   | D/V  |
| 1.12.3 | 의심스러운 접근 이벤트에 대해 경고가 생성되고 신속하게 조사되는지 확인합니다.                           |   2   | D/V  |

## C1.13 데이터 보유 및 만료 정책

불필요한 데이터 저장을 최소화하기 위해 데이터 보존 기간을 정의하고 적용합니다.

|   #    | Description                                   | Level | Role |
| :----: | --------------------------------------------- | :---: | :--: |
| 1.13.1 | 모든 학습 데이터 세트에 대해 명시적인 보존 기간이 정의되어 있는지 확인하십시오. |   1   | D/V  |
| 1.13.2 | 데이터셋이 수명 종료 시 자동으로 만료, 삭제되거나 삭제 검토되는지 확인하십시오. |   2   | D/V  |
| 1.13.3 | 보존 및 삭제 작업이 기록되고 감사 가능함을 확인하십시오.              |   2   | D/V  |

## C1.14 규제 및 관할권 준수

모든 학습 데이터가 해당 법률 및 규정을 준수하는지 확인하십시오.

|   #    | Description                                                     | Level | Role |
| :----: | --------------------------------------------------------------- | :---: | :--: |
| 1.14.1 | 모든 데이터 세트에 대해 데이터 거주지 및 국경 간 전송 요구 사항이 식별되고 준수되는지 확인하십시오.       |   2   | D/V  |
| 1.14.2 | 섹터별 규제(예: 의료, 금융)가 데이터 처리에서 식별되고 충족되는지 확인하십시오.                  |   2   | D/V  |
| 1.14.3 | 관련 개인정보 보호법(예: GDPR, CCPA) 준수가 문서화되어 있으며 정기적으로 검토되고 있는지 확인하십시오. |   2   | D/V  |

## C1.15 데이터 워터마킹 및 지문인식

독점적이거나 민감한 데이터의 무단 재사용 또는 유출을 감지합니다.

| :----: | ----------------------------------------------------------- | :---: | :--: |

| 1.15.2 | 워터마킹/핑거프린팅 방법이 편향이나 개인정보 침해 위험을 유발하지 않는지 검증하십시오.            |   3   | D/V  |
| 1.15.3 | 워터마크가 삽입된 데이터의 무단 재사용 또는 유출을 감지하기 위해 주기적인 점검이 수행되는지 확인하십시오. |   3   | D/V  |

| 1.16.3 | 데이터 주체 권리 프로세스가 정기적으로 테스트되고 효과성을 위해 검토되는지 확인하십시오.         |   2   | D/V  |

| 1.17.2 | 영향 분석 결과가 문서화되고 관련 이해관계자에 의해 검토되었는지 확인합니다.                                       |   2   | D/V  |
| 1.17.3 | 새 버전이 허용할 수 없는 위험이나 역행을 초래할 경우에 대비하여 롤백 계획이 존재하는지 확인하십시오.                        |   2   | D/V  |
