# C12 모니터링, 로깅 및 이상 탐지

## 통제 목표

이 섹션은 위협을 탐지, 분류 및 학습할 수 있도록 모델과 기타 AI 구성 요소가 보는 것, 수행하는 작업 및 반환하는 내용을 실시간 및 포렌식 가시성으로 제공하기 위한 요구사항을 제공합니다.

## C12.1 요청 및 응답 로깅

|   #    | Description                                                                                                         | Level | Role |
| :----: | ------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 12.1.1 | 모든 사용자 프롬프트와 모델 응답이 적절한 메타데이터(예: 타임스탬프, 사용자 ID, 세션 ID, 모델 버전)와 함께 기록되는지 확인하십시오.                                     |   1   | D/V  |
| 12.1.2 | 로그가 적절한 보존 정책과 백업 절차를 갖춘 보안 및 접근 제어된 저장소에 저장되어 있는지 확인하십시오.                                                          |   1   | D/V  |
| 12.1.3 | 로그 저장 시스템이 로그에 포함된 민감한 정보를 보호하기 위해 저장 데이터 암호화와 전송 데이터 암호화를 구현했는지 확인하십시오.                                            |   1   | D/V  |
| 12.1.4 | 민감한 데이터가 프롬프트와 출력에서 자동으로 삭제되거나 마스킹되어 기록되기 전에 이를 확인하며, PII(개인 식별 정보), 자격 증명 및 독점 정보에 대한 구성 가능한 삭제 규칙을 설정할 수 있어야 합니다. |   1   | D/V  |
| 12.1.5 | 정책 결정 및 안전 필터링 조치가 콘텐츠 중재 시스템의 감사 및 디버깅을 가능하게 할 만큼 충분한 세부 정보와 함께 기록되는지 확인하십시오.                                      |   2   | D/V  |
| 12.1.6 | 예를 들어 암호화 서명 또는 쓰기 전용 저장소를 통해 로그 무결성이 보호되는지 확인하십시오.                                                                 |   2   | D/V  |

## C12.2 남용 감지 및 경고

|   #    | Description                                                                                   | Level | Role |
| :----: | --------------------------------------------------------------------------------------------- | :---: | :--: |
| 12.2.1 | 시스템이 시그니처 기반 탐지를 사용하여 알려진 탈옥 패턴, 프롬프트 인젝션 시도 및 적대적 입력을 감지하고 경고하는지 확인합니다.                      |   1   | D/V  |
| 12.2.2 | 시스템이 표준 로그 형식 및 프로토콜을 사용하여 기존 보안 정보 및 이벤트 관리(SIEM) 플랫폼과 통합되는지 확인하십시오.                         |   1   | D/V  |
| 12.2.3 | 강화된 보안 이벤트에 모델 식별자, 신뢰도 점수, 안전 필터 결정과 같은 AI 특화 컨텍스트가 포함되어 있는지 확인하세요.                          |   2   | D/V  |
| 12.2.4 | 행동 이상 탐지가 비정상적인 대화 패턴, 과도한 재시도 시도 또는 체계적인 탐색 행동을 식별하는지 확인하십시오.                                |   2   | D/V  |
| 12.2.5 | 실시간 경고 메커니즘이 잠재적인 정책 위반 또는 공격 시도가 감지될 때 보안 팀에 알리는지 확인하십시오.                                    |   2   | D/V  |
| 12.2.6 | AI 고유의 위협 패턴을 탐지하기 위해 맞춤 규칙이 포함되어 있는지 확인하십시오. 여기에는 조직적인 탈옥 시도, 프롬프트 인젝션 캠페인, 모델 추출 공격이 포함됩니다. |   2   | D/V  |
| 12.2.7 | 자동화된 사고 대응 워크플로가 손상된 모델을 격리하고 악성 사용자를 차단하며 중요한 보안 이벤트를 에스컬레이션할 수 있는지 확인하십시오.                  |   3   | D/V  |

## C12.3 모델 드리프트 감지

|   #    | Description                                                                   | Level | Role |
| :----: | ----------------------------------------------------------------------------- | :---: | :--: |
| 12.3.1 | 시스템이 모델 버전 및 기간에 걸쳐 정확도, 신뢰도 점수, 지연 시간 및 오류율과 같은 기본 성능 지표를 추적하는지 확인하십시오.      |   1   | D/V  |
| 12.3.2 | 성능 지표가 사전에 정의된 저하 임계값을 초과하거나 기준선에서 크게 벗어날 때 자동 경고가 작동하는지 확인하십시오.              |   2   | D/V  |
| 12.3.3 | 환각 탐지 모니터가 모델 출력에 사실과 다르거나 일관성이 없거나 조작된 정보가 포함된 경우 이를 식별하고 플래그를 지정하는지 확인하십시오. |   2   | D/V  |

---

## C12.4 성능 및 동작 텔레메트리

|   #    | Description                                                                      | Level | Role |
| :----: | -------------------------------------------------------------------------------- | :---: | :--: |
| 12.4.1 | 요청 지연 시간, 토큰 소비량, 메모리 사용량 및 처리량을 포함한 운영 지표가 지속적으로 수집되고 모니터링되는지 확인하십시오.           |   1   | D/V  |
| 12.4.2 | 성공률과 실패율이 오류 유형 및 그 근본 원인 분류와 함께 추적되는지 확인하십시오.                                   |   1   | D/V  |
| 12.4.3 | 리소스 활용도 모니터링에 GPU/CPU 사용량, 메모리 사용량, 저장 공간 요구 사항이 포함되며 임계값 초과 시 경고가 발생하는지 확인하십시오. |   2   | D/V  |

## C12.5 AI 사고 대응 계획 및 실행

|   #    | Description                                                              | Level | Role |
| :----: | ------------------------------------------------------------------------ | :---: | :--: |
| 12.5.1 | 사고 대응 계획이 모델 손상, 데이터 변조, 적대적 공격 등 AI 관련 보안 이벤트를 구체적으로 다루고 있는지 검증하십시오.    |   1   | D/V  |
| 12.5.2 | 사건 대응 팀이 모델 동작과 공격 경로를 조사하기 위해 AI 전용 포렌식 도구와 전문 지식에 접근할 수 있는지 확인하십시오.    |   2   | D/V  |
| 12.5.3 | 사후 사고 분석에 모델 재학습 고려사항, 안전 필터 업데이트 및 교훈을 보안 통제에 통합하는 내용이 포함되어 있는지 확인하십시오. |   3   | D/V  |

## C12.5 AI 성능 저하 탐지

시간 경과에 따른 AI 모델 성능 및 품질 저하를 모니터링하고 감지합니다.

|   #    | Description                                                     | Level | Role |
| :----: | --------------------------------------------------------------- | :---: | :--: |
| 12.5.1 | 모델의 정확도, 정밀도, 재현율 및 F1 점수가 지속적으로 모니터링되고 기준 임계값과 비교되는지 확인하십시오.   |   1   | D/V  |
| 12.5.2 | 데이터 드리프트 감지는 모델 성능에 영향을 미칠 수 있는 입력 분포 변화를 모니터링하는지 확인하십시오.       |   1   | D/V  |
| 12.5.3 | 개념 변화 탐지는 입력과 예상 출력 간의 관계 변화를 식별하는지 확인하십시오.                     |   2   | D/V  |
| 12.5.4 | 성능 저하가 자동 경고를 유발하고 모델 재훈련 또는 교체 워크플로우를 시작하는지 확인하십시오.            |   2   | D/V  |
| 12.5.5 | 성능 저하의 근본 원인 분석이 성능 저하를 데이터 변화, 인프라 문제 또는 외부 요인과 연관시키는지 확인하십시오. |   3   |  V   |

## C12.6 DAG 시각화 및 워크플로우 보안

워크플로 시각화 시스템을 정보 유출 및 조작 공격으로부터 보호하십시오.

|   #    | Description                                                              | Level | Role |
| :----: | ------------------------------------------------------------------------ | :---: | :--: |
| 12.6.1 | DAG 시각화 데이터가 저장 또는 전송 전에 민감한 정보가 제거되어 세정되었는지 확인하십시오.                     |   1   | D/V  |
| 12.6.2 | 워크플로 시각화 접근 제어가 권한이 있는 사용자만 에이전트 결정 경로 및 추론 과정을 볼 수 있도록 하는지 확인합니다.       |   1   | D/V  |
| 12.6.3 | DAG 데이터 무결성이 암호화 서명 및 변조 방지 저장 메커니즘을 통해 보호되는지 확인합니다.                     |   2   | D/V  |
| 12.6.4 | 워크플로우 시각화 시스템이 조작된 노드 또는 엣지 데이터를 통한 인젝션 공격을 방지하기 위해 입력 검증을 구현하는지 확인하십시오. |   2   | D/V  |
| 12.6.5 | 시각화 시스템에 대한 서비스 거부 공격을 방지하기 위해 실시간 DAG 업데이트가 속도 제한되고 검증되는지 확인하십시오.       |   3   | D/V  |

## C12.7 능동적 보안 행동 모니터링

사전 에이전트 행동 분석을 통한 보안 위협 탐지 및 방지.

|   #    | Description                                             | Level | Role |
| :----: | ------------------------------------------------------- | :---: | :--: |
| 12.7.1 | 실행 전에 위험 평가 통합과 함께 선제적 에이전트 행동이 보안 검증되었는지 확인하십시오.       |   1   | D/V  |
| 12.7.2 | 자율적 주도권 트리거에는 보안 컨텍스트 평가 및 위협 환경 평가가 포함되는지 확인하십시오.      |   2   | D/V  |
| 12.7.3 | 사전 예방적 행동 패턴이 잠재적인 보안 영향 및 의도하지 않은 결과에 대해 분석되었는지 검증합니다. |   2   | D/V  |
| 12.7.4 | 보안에 중요한 사전 조치가 명시적인 승인 절차와 감사 기록을 필요로 하는지 확인하십시오.       |   3   | D/V  |
| 12.7.5 | 행동 이상 탐지가 침해를 나타낼 수 있는 사전 대응 에이전트 패턴의 편차를 식별하는지 확인합니다.  |   3   | D/V  |
