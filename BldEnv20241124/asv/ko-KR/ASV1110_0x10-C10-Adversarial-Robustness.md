# 10 적대적 견고성 및 프라이버시 방어

## 제어 목표

AI 모델이 회피, 추론, 추출 또는 오염 공격에 직면했을 때 신뢰할 수 있고, 개인정보를 보호하며, 남용에 강하도록 보장하십시오.

---

## 10.1 모델 정렬 및 안전성

유해하거나 정책을 위반하는 출력에 대비하십시오.

|   #    | Description                                                             | Level | Role |
| :----: | ----------------------------------------------------------------------- | :---: | :--: |
| 10.1.1 | 정렬 테스트 스위트(레드팀 프롬프트, 탈옥 탐지, 금지된 콘텐츠)가 버전 관리되고 모든 모델 릴리스 시 실행되는지 확인하십시오. |   1   | D/V  |
| 10.1.2 | 거부 및 안전 완성 가드레일이 제대로 적용되었는지 확인하십시오.                                     |   1   |  D   |
| 10.1.3 | 자동 평가자가 유해 콘텐츠 비율을 측정하고 설정된 임계값을 초과하는 퇴보를 표시하는지 확인하세요.                  |   2   | D/V  |
| 10.1.4 | 카운터-제일브레이크 훈련이 문서화되어 있으며 재현 가능함을 검증하십시오.                                |   2   |  D   |
| 10.1.5 | 형식적인 정책 준수 증명서나 인증된 모니터링이 주요 영역을 커버하는지 확인하십시오.                          |   3   |  V   |

---

## 10.2 적대적 예제 강인화

조작된 입력에 대한 복원력을 높이십시오. 강력한 적대적 훈련과 벤치마크 점수가 현재 최선의 방법입니다.

|   #    | Description                                                | Level | Role |
| :----: | ---------------------------------------------------------- | :---: | :--: |
| 10.2.1 | 프로젝트 저장소에 재현 가능한 시드를 사용하는 적대적 학습 구성이 포함되어 있는지 확인하세요.       |   1   |  D   |
| 10.2.2 | 적대적 예제 탐지가 운영 파이프라인에서 차단 경고를 발생시키는지 확인하십시오.                |   2   | D/V  |
| 10.2.4 | 인증된 강건성 증명 또는 구간 경계 인증서가 최소한 가장 중요한 상위 클래스들을 포함하는지 확인하십시오. |   3   |  V   |
| 10.2.5 | 회귀 테스트가 적응형 공격을 사용하여 측정 가능한 견고성 손실이 없음을 확인하는지 검증하십시오.      |   3   |  V   |

---

## 10.3 멤버십 추론 완화

학습 데이터에 기록이 포함되었는지 여부를 결정하는 능력을 제한하세요. 차등 개인정보 보호 및 신뢰도 점수 마스킹이 가장 효과적인 알려진 방어 수단으로 남아 있습니다.

|   #    | Description                                                 | Level | Role |
| :----: | ----------------------------------------------------------- | :---: | :--: |
| 10.3.1 | 쿼리별 엔트로피 정규화 또는 온도 조정이 과도한 확신 예측을 줄이는지 확인하십시오.              |   1   |  D   |
| 10.3.2 | 민감한 데이터셋에 대해 ε-경계 차등 프라이버시 최적화를 사용하는지 확인하십시오.               |   2   |  D   |
| 10.3.3 | 공격 시뮬레이션(섀도우 모델 또는 블랙박스)이 보류된 데이터에서 공격 AUC ≤ 0.60임을 확인하십시오. |   2   |  V   |

---

## 10.4 모델 반전 저항성

개인 속성 재구성을 방지하십시오. 최근 조사들은 출력 자르기와 차분 개인정보보호(DP) 보장을 실용적인 방어 수단으로 강조합니다.

|   #    | Description                                                 | Level | Role |
| :----: | ----------------------------------------------------------- | :---: | :--: |
| 10.4.1 | 민감한 속성이 절대 직접 출력되지 않도록 확인하십시오; 필요한 경우 버킷 또는 일방향 변환을 사용하십시오. |   1   |  D   |
| 10.4.2 | 동일한 주체로부터 반복되는 적응형 쿼리가 쿼리 속도 제한에 의해 조절되는지 확인하십시오.           |   1   | D/V  |
| 10.4.3 | 모델이 개인정보 보호 노이즈로 훈련되었는지 확인하십시오.                             |   2   |  D   |

---

## 10.5 모델 추출 방어

무단 복제를 탐지하고 방지합니다. 워터마킹과 쿼리 패턴 분석을 권장합니다.

|   #    | Description                                                                  | Level | Role |
| :----: | ---------------------------------------------------------------------------- | :---: | :--: |
| 10.5.1 | 추론 게이트웨이가 모델의 기억 임계값에 맞춰 글로벌 및 API 키별 속도 제한을 강제하는지 확인하십시오.                   |   1   |  D   |
| 10.5.2 | 쿼리 엔트로피와 입력 복수성 통계가 자동화된 추출 탐지기에 입력된다는 것을 확인하십시오.                            |   2   | D/V  |
| 10.5.3 | 취약하거나 확률적인 워터마크가 의심되는 복제본에 대해 ≤ 1,000 쿼리 내에서 p < 0.01로 증명될 수 있는지 확인하십시오.     |   2   |  V   |
| 10.5.4 | 워터마크 키와 트리거 세트가 하드웨어 보안 모듈에 저장되며 매년 교체되는지 확인하십시오.                            |   3   |  D   |
| 10.5.5 | 추출 경고(extraction-alert) 이벤트에 문제가 되는 쿼리가 포함되어 있고 사건 대응 플레이북과 통합되어 있는지 확인하십시오. |   3   |  V   |

## 10.6 추론 시점의 오염 데이터 탐지

백도어가 삽입되었거나 오염된 입력을 식별하고 무력화합니다.

|   #    | Description                                             | Level | Role |
| :----: | ------------------------------------------------------- | :---: | :--: |
| 10.6.1 | 모델 추론 전에 입력이 이상 탐지기(예: STRIP, 일관성 점수 측정)를 통과하는지 확인하십시오. |   1   |  D   |
| 10.6.2 | 감지기 임계값이 깨끗한/오염된 검증 세트에서 조정되어 5% 미만의 오탐률을 달성하도록 확인하십시오. |   1   |  V   |
| 10.6.3 | 오염된 것으로 표시된 입력이 소프트 차단 및 인간 검토 워크플로를 유발하는지 확인하십시오.      |   2   |  D   |
| 10.6.4 | 탐지기가 적응형, 트리거 없는 백도어 공격에 대해 스트레스 테스트되었는지 확인하십시오.        |   2   |  V   |
| 10.6.5 | 탐지 효율성 지표가 기록되고 주기적으로 최신 위협 인텔리전스와 함께 재평가되는지 확인합니다.     |   3   |  D   |

## 10.7 동적 보안 정책 적응

위협 인텔리전스 및 행동 분석을 기반으로 하는 실시간 보안 정책 업데이트.

|   #    | Description                                                     | Level | Role |
| :----: | --------------------------------------------------------------- | :---: | :--: |
| 10.7.1 | 보안 정책이 정책 버전 무결성을 유지하면서 에이전트 재시작 없이 동적으로 업데이트될 수 있는지 확인하십시오.    |   1   | D/V  |
| 10.7.2 | 정책 업데이트가 권한이 있는 보안 담당자에 의해 암호화 서명되고 적용 전에 검증되었는지 확인하십시오.        |   2   | D/V  |
| 10.7.3 | 동적 정책 변경 사항이 정당성, 승인 체인 및 롤백 절차를 포함한 전체 감사 기록과 함께 기록되는지 확인하십시오. |   2   | D/V  |
| 10.7.4 | 적응형 보안 메커니즘이 위험 상황과 행동 패턴에 따라 위협 탐지 민감도를 조정하는지 확인하십시오.          |   3   | D/V  |
| 10.7.5 | 정책 적응 결정이 설명 가능하며 보안 팀 검토를 위한 증거 기록을 포함하는지 확인하십시오.              |   3   | D/V  |

## 10.8 리플렉션 기반 보안 분석

에이전트 자기성찰 및 메타인지 분석을 통한 보안 검증.

|   #    | Description                                                | Level | Role |
| :----: | ---------------------------------------------------------- | :---: | :--: |
| 10.8.1 | 에이전트 반영 메커니즘에 결정과 행동에 대한 보안 중심의 자기 평가가 포함되어 있는지 확인합니다.     |   1   | D/V  |
| 10.8.2 | 반사 출력이 적대적인 입력에 의한 자기 평가 메커니즘 조작을 방지하기 위해 검증되는지 확인하십시오.    |   2   | D/V  |
| 10.8.3 | 메타인지적 보안 분석이 에이전트 추론 과정에서 잠재적인 편향, 조작 또는 침해를 식별하는지 확인하십시오. |   2   | D/V  |
| 10.8.4 | 리플렉션 기반 보안 경고가 향상된 모니터링 및 잠재적인 인적 개입 워크플로를 트리거하는지 확인하십시오.  |   3   | D/V  |
| 10.8.5 | 보안 성찰로부터의 연속 학습이 정상 기능을 저해하지 않으면서 위협 탐지를 향상시키는지 검증하십시오.    |   3   | D/V  |

## 10.9 진화 및 자기개선 보안

자기 수정 및 진화가 가능한 에이전트 시스템에 대한 보안 통제.

|   #    | Description                                                                             | Level | Role |
| :----: | --------------------------------------------------------------------------------------- | :---: | :--: |
| 10.9.1 | 자기 수정 기능이 공식 검증 경계 내의 지정된 안전 영역에만 제한되어 있는지 확인하십시오.                                      |   1   | D/V  |
| 10.9.2 | 진화 제안이 구현 전에 보안 영향 평가를 거치는지 확인하십시오.                                                     |   2   | D/V  |
| 10.9.3 | 자기 개선 메커니즘에 무결성 검증을 포함한 롤백 기능이 포함되어 있는지 확인하십시오.                                         |   2   | D/V  |
| 10.9.4 | 메타러닝 보안이 개선 알고리즘의 적대적 조작을 방지하는지 검증하십시오.                                                 |   3   | D/V  |
| 10.9.5 | 재귀적 자기개선(recursive self-improvement)이 수렴에 대한 수학적 증명을 통해 형식적인 안전 제약 내에서 제한된다는 것을 검증하십시오. |   3   | D/V  |
