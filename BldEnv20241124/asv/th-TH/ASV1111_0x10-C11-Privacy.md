# 11 การปกป้องความเป็นส่วนตัวและการจัดการข้อมูลส่วนบุคคล

## วัตถุประสงค์การควบคุม

รักษาการรับประกันความเป็นส่วนตัวอย่างเข้มงวดตลอดวงจรชีวิตของ AI ทั้งการเก็บรวบรวม การฝึกอบรม การสรุปผล และการตอบสนองต่อเหตุการณ์ เพื่อให้ข้อมูลส่วนบุคคลถูกประมวลผลเฉพาะเมื่อได้รับความยินยอมอย่างชัดเจน โดยมีกำหนดขอบเขตขั้นต่ำที่จำเป็น การลบข้อมูลที่พิสูจน์ได้ และการรับประกันความเป็นส่วนตัวอย่างเป็นทางการ

---

## 11.1 การทำให้ไม่ระบุตัวตนและการลดข้อมูลให้น้อยที่สุด

|   #    | Description                                                                                                                                     | Level | Role |
| :----: | ----------------------------------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.1.1 | ตรวจสอบให้แน่ใจว่าตัวระบุโดยตรงและตัวระบุใกล้เคียงถูกลบหรือแฮชแล้ว                                                                              |   1   | D/V  |
| 11.1.2 | ตรวจสอบให้อัตโนมัติวัดค่า k-นิรนาม/l-ความหลากหลาย และแจ้งเตือนเมื่อเกณฑ์ต่ำกว่านโยบาย                                                           |   2   | D/V  |
| 11.1.3 | ตรวจสอบให้แน่ใจว่ารายงานความสำคัญของฟีเจอร์ในโมเดลพิสูจน์ว่าไม่มีการรั่วไหลของตัวระบุเกิน ε = 0.01 ของสารสนเทศเชิงร่วม (mutual information)     |   2   |  V   |
| 11.1.4 | ตรวจสอบให้แน่ใจว่าหลักฐานอย่างเป็นทางการหรือการรับรองข้อมูลสังเคราะห์แสดงความเสี่ยงในการระบุซ้ำ ≤ 0.05 แม้ในกรณีที่มีการโจมตีแบบเชื่อมโยงข้อมูล |   3   |  V   |

## 11.2 สิทธิ์ในการลบข้อมูลและการบังคับใช้การลบข้อมูล

|   #    | Description                                                                                                                                                                  | Level | Role |
| :----: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.2.1 | ตรวจสอบให้แน่ใจว่าคำขอลบข้อมูลของผู้ใช้จะถูกส่งต่อไปยังชุดข้อมูลดิบ, จุดตรวจสอบ, การฝัง, บันทึก และสำรองข้อมูลภายในข้อตกลงระดับการให้บริการที่น้อยกว่า 30 วัน                |   1   | D/V  |
| 11.2.2 | ตรวจสอบว่า "การลบความรู้ของเครื่อง" (machine-unlearning) ได้ทำการฝึกซ้ำหรือการลบโดยประมาณอย่างแท้จริงด้วยอัลกอริทึมการลบที่ได้รับการรับรอง (certified unlearning algorithms) |   2   |  D   |
| 11.2.3 | ตรวจสอบให้แน่ใจว่าการประเมินแบบจำลองเงาพิสูจน์ว่าบันทึกที่ลืมไปมีอิทธิพลต่อผลลัพธ์น้อยกว่า 1% หลังจากกระบวนการลบความทรงจำ                                                    |   2   |  V   |
| 11.2.4 | ตรวจสอบให้แน่ใจว่าเหตุการณ์การลบถูกบันทึกอย่างถาวรและสามารถตรวจสอบได้สำหรับหน่วยงานกำกับดูแล                                                                                 |   3   |  V   |

## 11.3 การป้องกันความเป็นส่วนตัวเชิงต่างค์

|   #    | Description                                                                                        | Level | Role |
| :----: | -------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.3.1 | ตรวจสอบให้แน่ใจว่าแดชบอร์ดการนับการสูญเสียความเป็นส่วนตัวแจ้งเตือนเมื่อค่ารวม ε เกินเกณฑ์นโยบาย    |   2   | D/V  |
| 11.3.2 | ตรวจสอบให้แน่ใจว่าการตรวจสอบความเป็นส่วนตัวแบบกล่องดำประเมินค่า ε̂ ได้ภายใน 10% ของค่าที่ประกาศไว้ |   2   |  V   |
| 11.3.3 | ตรวจสอบให้แน่ใจว่าการพิสูจน์ทางการครอบคลุมการปรับแต่งหลังการฝึกทั้งหมดและการฝังข้อมูล              |   3   |  V   |

## 11.4 การจำกัดวัตถุประสงค์และการป้องกันการลุกลามขอบเขต

|   #    | Description                                                                                                                                                          | Level | Role |
| :----: | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.4.1 | ตรวจสอบให้แน่ใจว่าชุดข้อมูลและจุดตรวจสอบโมเดลทุกชุดมีป้ายวัตถุประสงค์ที่อ่านโดยเครื่องได้ ซึ่งสอดคล้องกับความยินยอมเดิม                                              |   1   |  D   |
| 11.4.2 | ตรวจสอบให้แน่ใจว่าตัวตรวจสอบเวลาทำงานสามารถตรวจจับคำค้นหาที่ไม่สอดคล้องกับวัตถุประสงค์ที่ประกาศและกระตุ้นการปฏิเสธแบบนุ่มนวล                                         |   1   | D/V  |
| 11.4.3 | ตรวจสอบให้แน่ใจว่านโยบายแบบรหัส (policy-as-code) จะป้องกันไม่ให้มีการปรับใช้โมเดลซ้ำในโดเมนใหม่โดยไม่มีการตรวจสอบ DPIA (การประเมินผลกระทบด้านการปกป้องข้อมูลพื้นฐาน) |   3   |  D   |
| 11.4.4 | ตรวจสอบให้แน่ใจว่าหลักฐานการติดตามอย่างเป็นทางการแสดงให้เห็นว่วงจรชีวิตของข้อมูลส่วนบุคคลทุกขั้นตอนยังคงอยู่ภายในขอบเขตที่ได้รับความยินยอมแล้ว                       |   3   |  V   |

## 11.5 การจัดการความยินยอมและการติดตามฐานทางกฎหมาย

|   #    | Description                                                                                                                               | Level | Role |
| :----: | ----------------------------------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.5.1 | ตรวจสอบให้แน่ใจว่าแพลตฟอร์มการจัดการความยินยอม (CMP) บันทึกสถานะการยินยอม วัตถุประสงค์ และระยะเวลาการเก็บรักษาข้อมูลตามแต่ละเจ้าของข้อมูล |   1   | D/V  |
| 11.5.2 | ตรวจสอบให้แน่ใจว่า API เปิดเผยโทเค็นการยินยอม; โมเดลต้องตรวจสอบขอบเขตของโทเค็นก่อนการคาดการณ์ข้อมูล                                       |   2   |  D   |
| 11.5.3 | ตรวจสอบให้แน่ใจว่าการปฏิเสธหรือถอนความยินยอมจะหยุดกระบวนการประมวลผลภายใน 24 ชั่วโมง                                                       |   2   | D/V  |

## 11.6 การเรียนรู้แบบรวมกลุ่มพร้อมการควบคุมความเป็นส่วนตัว

|   #    | Description                                                                                                    | Level | Role |
| :----: | -------------------------------------------------------------------------------------------------------------- | :---: | :--: |
| 11.6.1 | ตรวจสอบให้แน่ใจว่าการอัปเดตของลูกค้าใช้การเพิ่มเสียงความเป็นส่วนตัวเชิงต่างระดับภายในเครื่องก่อนการรวมข้อมูล   |   1   |  D   |
| 11.6.2 | ตรวจสอบให้แน่ใจว่าเมตริกการฝึกอบรมมีความเป็นส่วนตัวเชิงแตกต่างและไม่เปิดเผยผลขาดทุนของลูกค้าแต่ละรายโดยเด็ดขาด |   2   | D/V  |
| 11.6.3 | ตรวจสอบให้แน่ใจว่าการรวมข้อมูลที่ต้านต่อการโจมตีด้วยพิษ (เช่น Krum/Trimmed-Mean) ถูกเปิดใช้งานแล้ว             |   2   |  V   |
| 11.6.4 | ตรวจสอบให้แน่ใจว่าพิสูจน์อย่างเป็นทางการแสดงงบประมาณ ε รวมโดยมีการสูญเสียวัตถุประสงค์น้อยกว่า 5 หน่วย          |   3   |  V   |
