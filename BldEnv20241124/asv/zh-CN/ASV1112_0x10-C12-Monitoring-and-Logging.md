# C12 监控、日志记录与异常检测

## 控制目标

本节提供了实现对模型及其他人工智能组件所观察、执行和返回内容的实时及取证可见性的要求，以便能够检测、分类和学习威胁。

## C12.1 请求与响应日志记录

|   #    | Description                                                   | Level | Role |
| :----: | ------------------------------------------------------------- | :---: | :--: |
| 12.1.1 | 验证所有用户提示和模型响应是否带有适当的元数据（例如时间戳、用户ID、会话ID、模型版本）进行日志记录。          |   1   | D/V  |
| 12.1.2 | 验证日志是否存储在安全的、访问受控的存储库中，且具备适当的保留策略和备份程序。                       |   1   | D/V  |
| 12.1.3 | 验证日志存储系统是否在静态和传输中实现了加密，以保护日志中包含的敏感信息。                         |   1   | D/V  |
| 12.1.4 | 验证提示和输出中的敏感数据在记录之前是否已自动屏蔽或遮盖，并且能够针对个人身份信息（PII）、凭证和专有信息配置屏蔽规则。 |   1   | D/V  |
| 12.1.5 | 验证政策决策和安全过滤操作是否被以足够详细的方式记录，以支持内容审核系统的审计和调试。                   |   2   | D/V  |
| 12.1.6 | 验证日志完整性是否通过例如加密签名或写入即不可更改的存储进行保护。                             |   2   | D/V  |

---

## C12.2 滥用检测与警报

|   #    | Description                                           | Level | Role |
| :----: | ----------------------------------------------------- | :---: | :--: |
| 12.2.1 | 验证系统是否通过基于签名的检测识别并警报已知的越狱模式、提示注入尝试和对抗性输入。             |   1   | D/V  |
| 12.2.2 | 验证系统是否使用标准日志格式和协议与现有的安全信息和事件管理（SIEM）平台集成。             |   1   | D/V  |
| 12.2.3 | 验证已增强的安全事件是否包含特定于人工智能的上下文，例如模型标识符、置信度分数和安全过滤器决策。      |   2   | D/V  |
| 12.2.4 | 验证行为异常检测是否能够识别异常的对话模式、过多的重试尝试或系统性的探测行为。               |   2   | D/V  |
| 12.2.5 | 验证实时警报机制在检测到潜在的策略违规或攻击尝试时能通知安全团队。                     |   2   | D/V  |
| 12.2.6 | 验证是否包含自定义规则以检测特定于人工智能的威胁模式，包括协调的越狱尝试、提示注入攻击活动和模型提取攻击。 |   2   | D/V  |
| 12.2.7 | 验证自动化事件响应工作流程能够隔离受损模型、阻止恶意用户并升级关键安全事件。                |   3   | D/V  |

---

## C12.3 模型漂移检测

|   #    | Description                                   | Level | Role |
| :----: | --------------------------------------------- | :---: | :--: |
| 12.3.1 | 验证系统是否跟踪基本的性能指标，如准确率、置信度分数、延迟和错误率，涵盖模型版本和时间段。 |   1   | D/V  |
| 12.3.2 | 验证当性能指标超过预定义的降级阈值或显著偏离基线时，自动警报是否被触发。          |   2   | D/V  |
| 12.3.3 | 验证幻觉检测监测器是否能够识别并标记模型输出中包含事实错误、不一致或捏造信息的实例。    |   2   | D/V  |

---

## C12.4 性能与行为遥测

|   #    | Description                                      | Level | Role |
| :----: | ------------------------------------------------ | :---: | :--: |
| 12.4.1 | 验证操作指标，包括请求延迟、令牌消耗、内存使用和吞吐量，是否被持续收集和监控。          |   1   | D/V  |
| 12.4.2 | 验证成功率和失败率是否被跟踪，并且错误类型及其根本原因是否被分类。                |   1   | D/V  |
| 12.4.3 | 验证资源利用率监控是否包括 GPU/CPU 使用率、内存消耗和存储需求，并在阈值超限时发出警报。 |   2   | D/V  |

---

## C12.5 人工智能事件响应规划与执行

|   #    | Description                                  | Level | Role |
| :----: | -------------------------------------------- | :---: | :--: |
| 12.5.1 | 确认事件响应计划专门针对与人工智能相关的安全事件，包括模型被攻破、数据投毒和对抗性攻击。 |   1   | D/V  |
| 12.5.2 | 确保事件响应团队能够使用专门针对人工智能的取证工具和专业知识，以调查模型行为和攻击途径。 |   2   | D/V  |
| 12.5.3 | 确认事故后分析包括模型再训练的考虑、安全过滤器的更新以及将经验教训整合到安全控制中。   |   3   | D/V  |

---

## C12.5 AI性能退化检测

监控并检测人工智能模型性能和质量随时间的退化。

|   #    | Description                            | Level | Role |
| :----: | -------------------------------------- | :---: | :--: |
| 12.5.1 | 确认模型的准确率、精确率、召回率和F1分数被持续监控，并与基线阈值进行比较。 |   1   | D/V  |
| 12.5.2 | 验证数据漂移检测是否监测可能影响模型性能的输入分布变化。           |   1   | D/V  |
| 12.5.3 | 验证概念漂移检测是否能够识别输入与预期输出之间关系的变化。          |   2   | D/V  |
| 12.5.4 | 验证性能下降是否会触发自动警报并启动模型重新训练或更换工作流程。       |   2   | D/V  |
| 12.5.5 | 验证降级根本原因分析是否将性能下降与数据变化、基础设施问题或外部因素相关联。 |   3   |  V   |

## C12.6 有向无环图（DAG）可视化与工作流安全

保护工作流可视化系统免受信息泄露和篡改攻击。

|   #    | Description                                 | Level | Role |
| :----: | ------------------------------------------- | :---: | :--: |
| 12.6.1 | 验证有向无环图（DAG）可视化数据在存储或传输前已进行清理，以去除敏感信息。      |   1   | D/V  |
| 12.6.2 | 验证工作流可视化访问控制，确保只有授权用户才能查看代理的决策路径和推理轨迹。      |   1   | D/V  |
| 12.6.3 | 验证DAG数据完整性是否通过加密签名和防篡改存储机制得到保护。             |   2   | D/V  |
| 12.6.4 | 验证工作流可视化系统是否实现了输入验证，以防止通过精心构造的节点或边数据进行注入攻击。 |   2   | D/V  |
| 12.6.5 | 验证实时DAG更新是否有限速，并经过验证以防止对可视化系统的拒绝服务攻击。       |   3   | D/V  |

---

## C12.7 主动安全行为监控

通过主动代理行为分析实现安全威胁的检测与防范。

|   #    | Description                              | Level | Role |
| :----: | ---------------------------------------- | :---: | :--: |
| 12.7.1 | 在执行前，通过风险评估集成验证主动代理行为的安全性。               |   1   | D/V  |
| 12.7.2 | 验证自主主动触发条件是否包括安全上下文评估和威胁态势评估。            |   2   | D/V  |
| 12.7.3 | 验证主动行为模式是否已被分析以识别潜在的安全影响和意外后果。           |   2   | D/V  |
| 12.7.4 | 验证安全关键的主动措施需要有明确的审批链和审计追踪。               |   3   | D/V  |
| 12.7.5 | 验证行为异常检测是否能够识别主动代理模式中的偏差，这些偏差可能表明存在安全漏洞。 |   3   | D/V  |
