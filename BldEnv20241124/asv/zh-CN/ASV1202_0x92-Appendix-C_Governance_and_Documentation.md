# 附录C：人工智能安全治理与文档

## 目标

本附录提供了建立组织结构、政策和流程的基本要求，以在整个系统生命周期内管理人工智能安全。

---

## AC.1 人工智能风险管理框架采纳

提供一个正式框架，以在系统生命周期内识别、评估和缓解特定于人工智能的风险。

|   #    | Description                       | Level | Role |
| :----: | --------------------------------- | :---: | :--: |
| AC.1.1 | 验证已记录并实施专门针对人工智能的风险评估方法。          |   1   | D/V  |
| AC.1.2 | 确认在人工智能生命周期的关键节点以及重大变更之前进行风险评估。   |   2   |  D   |
| AC.1.3 | 验证风险管理框架是否符合既定标准（例如，NIST AI RMF）。 |   3   | D/V  |

---

## AC.2 人工智能安全政策与程序

定义并执行组织标准，以确保人工智能的安全开发、部署和运行。

|   #    | Description                      | Level | Role |
| :----: | -------------------------------- | :---: | :--: |
| AC.2.1 | 验证是否存在已记录的人工智能安全策略。              |   1   | D/V  |
| AC.2.2 | 确保政策至少每年审查和更新一次，并在重大威胁环境变化后进行更新。 |   2   |  D   |
| AC.2.3 | 验证政策是否涵盖所有AISVS类别及适用的监管要求。       |   3   | D/V  |

---

## AC.3 人工智能安全的角色与职责

在整个组织中建立明确的人工智能安全责任制。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| AC.3.1 | 验证人工智能安全角色和职责是否有文档记录。           |   1   | D/V  |
| AC.3.2 | 确认相关人员具备适当的安全专业知识。              |   2   |  D   |
| AC.3.3 | 确认已为高风险人工智能系统建立人工智能伦理委员会或治理委员会。 |   3   | D/V  |

---

## AC.4 伦理人工智能指南执行

确保人工智能系统按照既定的伦理原则运行。

|   #    | Description             | Level | Role |
| :----: | ----------------------- | :---: | :--: |
| AC.4.1 | 验证是否存在人工智能开发和部署的伦理指南。   |   1   | D/V  |
| AC.4.2 | 验证是否已建立机制以检测和报告伦理违规行为。  |   2   |  D   |
| AC.4.3 | 验证已部署的人工智能系统是否定期进行伦理审查。 |   3   | D/V  |

---

## AC.5 人工智能法规合规监测

保持对不断变化的人工智能法规的关注和遵守。

|   #    | Description                | Level | Role |
| :----: | -------------------------- | :---: | :--: |
| AC.5.1 | 验证是否存在识别适用人工智能法规的流程。       |   1   | D/V  |
| AC.5.2 | 验证是否已评估所有监管要求的合规性。         |   2   |  D   |
| AC.5.3 | 确认监管变化能够及时触发对人工智能系统的审查和更新。 |   3   | D/V  |
