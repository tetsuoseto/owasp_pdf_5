# C1 训练数据治理与偏差管理

## 控制目标

训练数据必须以保持来源、 安全性、质量和公平性的方式进行获取、处理和维护。这样做既履行了法律义务，又减少了整个人工智能生命周期中出现偏见、中毒或隐私泄露的风险。

---

## C1.1 训练数据来源

保持所有数据集的可验证清单，仅接受可信来源，并记录每次更改以确保可审计性。

|   #   | Description                                                      | Level | Role |
| :---: | ---------------------------------------------------------------- | :---: | :--: |
| 1.1.1 | 验证是否维护了每个训练数据源的最新清单，包括来源、负责人/所有者、许可、收集方法、预期使用限制和处理历史。            |   1   | D/V  |
| 1.1.2 | 验证仅允许经过质量、代表性、道德来源和许可证合规性审核的数据集，以减少中毒、内嵌偏见和知识产权侵权的风险。            |   1   | D/V  |
| 1.1.3 | 验证是否强制执行数据最小化，以排除不必要或敏感的属性。                                      |   1   | D/V  |
| 1.1.4 | 验证所有数据集的更改均需经过记录的审批流程。                                           |   2   | D/V  |
| 1.1.5 | 通过审核者交叉核对或达成共识，确保标注/注释质量。                                        |   2   | D/V  |
| 1.1.6 | 确保为重要的训练数据集维护“数据卡”或“数据集说明书”，详细说明其特征、动机、组成、采集过程、预处理以及推荐/不推荐的使用方式。 |   2   | D/V  |

---

## C1.2 训练数据的安全性与完整性

限制访问，对静态数据和传输中的数据进行加密，并验证完整性以阻止篡改或盗窃。

|   #   | Description                                                               | Level | Role |
| :---: | ------------------------------------------------------------------------- | :---: | :--: |
| 1.2.1 | 验证访问控制是否保护存储和数据管道。                                                        |   1   | D/V  |
| 1.2.2 | 确保数据集在传输过程中加密，并且对于所有敏感或个人身份信息（PII），在静止状态下也进行加密，采用行业标准的加密算法和密钥管理规范。        |   2   | D/V  |
| 1.2.3 | 验证在存储和传输过程中使用了加密哈希或数字签名以确保数据完整性，并且应用了自动异常检测技术以防止未经授权的修改或损坏，包括有针对性的数据投毒尝试。 |   2   | D/V  |
| 1.2.4 | 验证数据集版本是否受到跟踪以支持回滚。                                                       |   3   | D/V  |
| 1.2.5 | 验证过时数据是否已根据数据保留政策、监管要求安全清除或匿名处理，以缩小攻击面。                                   |   2   | D/V  |

---

## C1.3 表示的完整性与公平性

检测人口统计偏差并采取缓解措施，以确保模型的错误率在各群体间公平。

|   #   | Description                                                                                 | Level | Role |
| :---: | ------------------------------------------------------------------------------------------- | :---: | :--: |
| 1.3.1 | 验证数据集是否已针对表征不平衡和潜在偏见进行分析，涵盖法律保护属性（例如种族、性别、年龄）以及与模型应用领域相关的其他伦理敏感特征（例如社会经济状况、地点）。             |   1   | D/V  |
| 1.3.2 | 验证所识别的偏差是否通过有文档记录的策略得到缓解，例如重新平衡、有针对性的数据增强、算法调整（例如预处理、处理中、后处理技术）或重新加权，并评估缓解措施对公平性和整体模型性能的影响。 |   2   | D/V  |
| 1.3.3 | 验证训练后公平性指标是否已被评估和记录。                                                                        |   2   | D/V  |
| 1.3.4 | 验证生命周期偏差管理政策是否分配了负责人和审查频率。                                                                  |   3   | D/V  |

---

## C1.4 训练数据的标注质量、完整性与安全性

使用加密保护标签，并对关键类别要求双重审核。

|   #   | Description                                                        | Level | Role |
| :---: | ------------------------------------------------------------------ | :---: | :--: |
| 1.4.1 | 通过明确的指导方针、审核人员交叉检查、一致性机制（例如监控标注者间的一致性），以及解决分歧的定义流程，确保标注/注释质量。      |   2   | D/V  |
| 1.4.2 | 验证是否对标签工件应用了加密哈希或数字签名，以确保其完整性和真实性。                                 |   2   | D/V  |
| 1.4.3 | 验证标签界面和平台是否实施了严格的访问控制，保持所有标签活动的防篡改审计日志，并防止未经授权的修改。                 |   2   | D/V  |
| 1.4.4 | 确保与安全性、保密性或公平性至关重要的标签（例如，识别有害内容、关键医疗发现）必须经过强制性的独立双重审查或等效的严格验证。     |   3   | D/V  |
| 1.4.5 | 验证根据数据最小化原则，对标签中无意捕获或必要存在的敏感信息（包括个人身份信息）进行了静态和传输中的编辑、假名化、匿名化或加密处理。 |   2   | D/V  |
| 1.4.6 | 确认标注指南和说明是全面的、具有版本控制的，并经过同行评审。                                     |   2   | D/V  |
| 1.4.7 | 确保数据模式（包括标签的模式）有明确定义，并且进行版本控制。                                     |   2   | D/V  |
| 1.8.2 | 验证外包或众包标注工作流程是否包含技术/程序上的保障措施，以确保数据的机密性、完整性、标签质量，并防止数据泄露。           |   2   | D/V  |

---

## C1.5 训练数据质量保证

结合自动验证、人工抽查和记录的修正措施，以确保数据集的可靠性。

|   #   | Description                                                                 | Level | Role |
| :---: | --------------------------------------------------------------------------- | :---: | :--: |
| 1.5.1 | 验证自动化测试是否在每次数据摄取或重大转换时捕获格式错误、空值和标签偏差。                                       |   1   |  D   |
| 1.5.2 | 验证失败的数据集是否被隔离并留有审计追踪记录。                                                     |   1   | D/V  |
| 1.5.3 | 验证领域专家的手工抽查覆盖了统计学上显著的样本（例如，≥1%或1000个样本，以更大者为准，或根据风险评估确定），以识别自动化未能发现的细微质量问题。 |   2   |  V   |
| 1.5.4 | 验证补救步骤已添加到溯源记录中。                                                            |   2   | D/V  |
| 1.5.5 | 验证质量门控会阻止未经批准的低质量数据集。                                                       |   2   | D/V  |

---

## C1.6 数据投毒检测

应用统计异常检测和隔离工作流程以阻止对抗性插入。

|   #   | Description                                                                  | Level | Role |
| :---: | ---------------------------------------------------------------------------- | :---: | :--: |
| 1.6.1 | 验证是否在数据摄取阶段和主要训练运行之前，对训练数据应用了异常检测技术（例如，统计方法、异常值检测、嵌入分析），以识别潜在的投毒攻击或非故意的数据损坏。 |   2   | D/V  |
| 1.6.2 | 确保被标记的样本在训练前触发人工审核。                                                          |   2   | D/V  |
| 1.6.3 | 验证结果是否反馈至模型的安全档案，并为持续的威胁情报提供信息。                                              |   2   |  V   |
| 1.6.4 | 确认检测逻辑已使用新的威胁情报进行更新。                                                         |   3   | D/V  |
| 1.6.5 | 验证在线学习管道是否监控分布漂移。                                                            |   3   | D/V  |
| 1.6.6 | 根据系统的风险概况和数据来源，验证是否考虑并实施了针对已知数据中毒攻击类型（例如标签翻转、后门触发器插入、影响实例攻击）的具体防御措施。         |   3   | D/V  |

---

## C1.7 用户数据删除与同意执行

在各数据集、备份和派生产物中尊重删除和撤回同意的请求。

|   #   | Description                                                            | Level | Role |
| :---: | ---------------------------------------------------------------------- | :---: | :--: |
| 1.7.1 | 验证删除工作流是否清除主数据和派生数据，并评估对模型的影响；同时评估受影响模型的影响，并在必要时进行处理（例如，通过再训练或重新校准）。   |   1   | D/V  |
| 1.7.2 | 验证是否已建立机制以跟踪和尊重用户对用于训练数据的同意范围和状态（包括撤回），并在将数据纳入新的训练过程或重大模型更新之前验证同意的有效性。 |   2   |  D   |
| 1.7.3 | 确认工作流程每年均经过测试并记录在案。                                                    |   2   |  V   |

---

## C1.8 供应链安全

审核外部数据提供者，并通过认证的加密通道验证数据完整性。

|   #   | Description                                                                      | Level | Role |
| :---: | -------------------------------------------------------------------------------- | :---: | :--: |
| 1.8.1 | 验证第三方数据供应商，包括预训练模型提供者和外部数据集供应商，在其数据或模型被集成之前，是否经过安全、隐私、伦理采购和数据质量的尽职调查。            |   2   | D/V  |
| 1.8.2 | 验证外部传输是否使用 TLS/身份验证和完整性检查。                                                       |   1   |  D   |
| 1.8.3 | 确保对高风险数据源（例如，来源不明的开源数据集、未经审核的供应商）进行加强审查，例如在敏感应用中使用前进行沙箱分析、全面的质量/偏见检查以及有针对性的投毒检测。 |   2   | D/V  |
| 1.8.4 | 确保从第三方获得的预训练模型在微调或部署之前，已经评估了其嵌入的偏见、潜在的后门、架构的完整性以及其原始训练数据的来源。                     |   3   | D/V  |

---

## C1.9 对抗样本检测

在训练阶段实施措施，例如对抗训练，以增强模型对抗对抗性样本的弹性。

|   #   | Description                                                         | Level | Role |
| :---: | ------------------------------------------------------------------- | :---: | :--: |
| 1.9.1 | 根据风险评估，验证针对相关模型是否实施并调整了适当的防御措施，如对抗性训练（使用生成的对抗样本）、带扰动输入的数据增强或鲁棒优化技术。 |   3   | D/V  |
| 1.9.2 | 验证如果使用了对抗性训练，是否对对抗性数据集的生成、管理和版本控制进行了记录和控制。                          |   2   | D/V  |
| 1.9.3 | 验证对抗鲁棒性训练对模型性能（针对干净和对抗输入）及公平性指标的影响是否进行了评估、记录和监控。                    |   3   | D/V  |
| 1.9.4 | 验证用于对抗训练和鲁棒性的策略是否定期审查和更新，以应对不断演变的对抗攻击技术。                            |   3   | D/V  |

---

## C1.10 数据血缘和可追溯性

跟踪每个数据点从来源到模型输入的完整流程，以便审计和事件响应。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.10.1 | 验证每个数据点的血统，包括所有的转换、增强和合并，是否被记录并且能够重建。 |   2   | D/V  |
| 1.10.2 | 验证血统记录是否不可更改，安全存储，并可供审计或调查访问。         |   2   | D/V  |
| 1.10.3 | 验证血统跟踪涵盖原始数据和合成数据。                    |   2   | D/V  |

---

## C1.11 合成数据管理

确保合成数据得到妥善管理、标注并进行风险评估。

|   #    | Description                               | Level | Role |
| :----: | ----------------------------------------- | :---: | :--: |
| 1.11.1 | 验证所有合成数据在整个流程中均有明确标注且与真实数据可区分。            |   2   | D/V  |
| 1.11.2 | 确保生成过程、参数及合成数据的预期用途均有文档记录。                |   2   | D/V  |
| 1.11.3 | 在使用合成数据进行训练之前，需验证其已针对偏见、隐私泄露和表现问题进行了风险评估。 |   2   | D/V  |

---

## C1.12 数据访问监控与异常检测

监控并警报对训练数据的异常访问，以检测内部威胁或数据外泄。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| 1.12.1 | 验证所有对训练数据的访问是否都有日志记录，包括用户、时间和操作。     |   2   | D/V  |
| 1.12.2 | 确认访问日志是否定期审查，以发现异常模式，如大量导出或来自新位置的访问。 |   2   | D/V  |
| 1.12.3 | 验证是否针对可疑访问事件生成了警报并及时进行了调查。           |   2   | D/V  |

---

## C1.13 数据保留与过期政策

定义并执行数据保留期限，以最大限度减少不必要的数据存储。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| 1.13.1 | 验证所有训练数据集是否明确规定了保存期限。           |   1   | D/V  |
| 1.13.2 | 验证数据集在生命周期结束时是否会自动过期、删除或被审查后删除。 |   2   | D/V  |
| 1.13.3 | 验证保留和删除操作是否被记录和可审计。             |   2   | D/V  |

---

## C1.14 法规与司法管辖合规性

确保所有训练数据符合适用的法律法规。

|   #    | Description                        | Level | Role |
| :----: | ---------------------------------- | :---: | :--: |
| 1.14.1 | 验证所有数据集的数据驻留和跨境传输要求是否已被识别并执行。      |   2   | D/V  |
| 1.14.2 | 确认已识别并处理特定行业法规（如医疗、金融）在数据处理中的要求。   |   2   | D/V  |
| 1.14.3 | 验证相关隐私法律（如GDPR、CCPA）的合规性已被记录并定期审查。 |   2   | D/V  |

---

## C1.15 数据水印与指纹识别

检测专有或敏感数据的未经授权的重复使用或泄露。

|   #    | Description                   | Level | Role |
| :----: | ----------------------------- | :---: | :--: |
| 1.15.1 | 在可行的情况下，验证数据集或子集是否带有水印或指纹标识。  |   3   | D/V  |
| 1.15.2 | 验证水印/指纹识别方法是否不会引入偏见或隐私风险。     |   3   | D/V  |
| 1.15.3 | 验证是否定期进行检查，以检测水印数据的未经授权重用或泄露。 |   3   | D/V  |

---

## C1.16 数据主体权利管理

支持数据主体权利，如访问、更正、限制和反对。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| 1.16.1 | 验证是否存在响应数据主体关于访问、更正、限制或反对请求的机制。 |   2   | D/V  |
| 1.16.2 | 验证请求是否在法律规定的时间范围内被记录、跟踪和完成。     |   2   | D/V  |
| 1.16.3 | 验证数据主体权利流程是否经过定期测试和审查以确保其有效性。   |   2   | D/V  |

---

## C1.17 数据集版本影响分析

在更新或替换数据集版本之前，评估数据集变化的影响。

|   #    | Description                             | Level | Role |
| :----: | --------------------------------------- | :---: | :--: |
| 1.17.1 | 在更新或替换数据集版本之前，确认已进行影响分析，涵盖模型性能、公平性和合规性。 |   2   | D/V  |
| 1.17.2 | 确认影响分析的结果已被相关利益相关者记录和审查。                |   2   | D/V  |
| 1.17.3 | 确认存在回滚计划，以防新版本引入不可接受的风险或回退问题。           |   2   | D/V  |

## C1.18 数据标注人员安全

确保参与数据标注人员的安全和完整性。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.18.1 | 确保所有参与数据标注的人员均经过背景审查并接受了数据安全和隐私方面的培训。 |   2   | D/V  |
| 1.18.2 | 确保所有注释人员都签署保密和不披露协议。                  |   2   | D/V  |
| 1.18.3 | 验证注释平台是否执行访问控制并监控内部威胁。                |   2   | D/V  |
