# 10 对抗鲁棒性与隐私防御

## 控制目标

确保在面对规避、推断、提取或投毒攻击时，人工智能模型保持可靠性、隐私保护和抗滥用能力。

---

## 10.1 模型对齐与安全

防止产生有害或违反政策的输出。

|   #    | Description                                      | Level | Role |
| :----: | ------------------------------------------------ | :---: | :--: |
| 10.1.1 | 验证对齐测试套件（红队提示、越狱探测、不允许的内容）是否进行了版本控制，并在每次模型发布时运行。 |   1   | D/V  |
| 10.1.2 | 验证拒绝和安全完成保护措施是否得到执行。                             |   1   |  D   |
| 10.1.3 | 验证自动评估器是否测量有害内容率，并标记超过设定阈值的回归情况。                 |   2   | D/V  |
| 10.1.4 | 验证反越狱训练是否有文档记录且可重复。                              |   2   |  D   |
| 10.1.5 | 验证正式的政策合规证明或认证监控是否涵盖关键领域。                        |   3   |  V   |

---

## 10.2 对抗样本加固

提高对操纵输入的抗干扰能力。稳健的对抗训练和基准评分是当前的最佳实践。

|   #    | Description                       | Level | Role |
| :----: | --------------------------------- | :---: | :--: |
| 10.2.1 | 验证项目仓库是否包含带有可复现种子的对抗训练配置。         |   1   |  D   |
| 10.2.2 | 验证对抗样本检测是否在生产流水线中触发阻断警报。          |   2   | D/V  |
| 10.2.4 | 验证认证的鲁棒性证明或区间界限证书是否至少涵盖了最关键的顶级类别。 |   3   |  V   |
| 10.2.5 | 验证回归测试使用自适应攻击以确认没有可测量的鲁棒性损失。      |   3   |  V   |

---

## 10.3 成员推断缓解措施

限制判断某条记录是否在训练数据中的能力。差分隐私和置信度评分屏蔽仍然是已知的最有效防御措施。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| 10.3.1 | 验证每个查询的熵正则化或温度缩放是否能减少过度自信的预测。        |   1   |  D   |
| 10.3.2 | 验证训练是否针对敏感数据集采用了ε-界限差分隐私优化。          |   2   |  D   |
| 10.3.3 | 验证攻击模拟（影子模型或黑盒）在保留数据上的攻击 AUC ≤ 0.60。 |   2   |  V   |

---

## 10.4 模型反演抵抗性

防止重建私有属性。近期调查强调输出截断和差分隐私保障作为实际防御措施。

|   #    | Description                   | Level | Role |
| :----: | ----------------------------- | :---: | :--: |
| 10.4.1 | 验证敏感属性从不被直接输出；在必要时，使用分桶或单向变换。 |   1   |  D   |
| 10.4.2 | 验证查询速率限制是否限制来自同一主体的重复自适应查询。   |   1   | D/V  |
| 10.4.3 | 验证模型是用隐私保护噪声训练的。              |   2   |  D   |

---

## 10.5 模型提取防御

检测和阻止未经授权的克隆。建议使用水印技术和查询模式分析。

|   #    | Description                                               | Level | Role |
| :----: | --------------------------------------------------------- | :---: | :--: |
| 10.5.1 | 验证推理网关是否执行全局和每个API密钥的速率限制，这些限制已调整至模型的记忆阈值。                |   1   |  D   |
| 10.5.2 | 验证查询熵和输入多样性统计数据是否输入自动提取检测器。                               |   2   | D/V  |
| 10.5.3 | 验证脆弱型或概率型水印是否能在对疑似克隆对象进行不超过1000次查询时，以p值小于0.01的显著性水平被证明有效。 |   2   |  V   |
| 10.5.4 | 验证水印密钥和触发集是否存储在硬件安全模块中，并且每年进行轮换。                          |   3   |  D   |
| 10.5.5 | 验证提取警报事件是否包含违规查询，并且是否已集成到事件响应剧本中。                         |   3   |  V   |

---

## 10.6 推理时的中毒数据检测

识别并中和带有后门或被投毒的输入。

|   #    | Description                            | Level | Role |
| :----: | -------------------------------------- | :---: | :--: |
| 10.6.1 | 验证输入在模型推理前通过异常检测器（例如，STRIP、一致性评分）。     |   1   |  D   |
| 10.6.2 | 验证检测器阈值是否在干净/已投毒的验证集上进行调优，以实现低于5%的误报率。 |   1   |  V   |
| 10.6.3 | 验证被标记为中毒的输入是否会触发软阻断和人工审核工作流程。          |   2   |  D   |
| 10.6.4 | 验证探测器是否通过自适应、无触发后门攻击进行了压力测试。           |   2   |  V   |
| 10.6.5 | 验证检测效能指标是否被记录，并定期使用最新的威胁情报进行重新评估。      |   3   |  D   |

---

## 10.7 动态安全策略适应

基于威胁情报和行为分析的实时安全策略更新。

|   #    | Description                                | Level | Role |
| :----: | ------------------------------------------ | :---: | :--: |
| 10.7.1 | 验证安全策略可以在不重启代理的情况下动态更新，同时保持策略版本的完整性。       |   1   | D/V  |
| 10.7.2 | 验证策略更新是否由授权的安全人员进行密码学签名，并在应用前进行验证。         |   2   | D/V  |
| 10.7.3 | 验证动态策略更改是否被记录，且包含完整的审计跟踪信息，包括理由、批准链条和回滚程序。 |   2   | D/V  |
| 10.7.4 | 验证自适应安全机制是否根据风险上下文和行为模式调整威胁检测的敏感度。         |   3   | D/V  |
| 10.7.5 | 确保策略调整决策具有可解释性，并包含供安全团队审查的证据记录。            |   3   | D/V  |

---

## 10.8 基于反射的安全分析

通过代理自我反思和元认知分析进行安全验证。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| 10.8.1 | 验证代理反思机制是否包括针对决策和行为的安全性自我评估。         |   1   | D/V  |
| 10.8.2 | 验证反射输出是否经过验证，以防止对抗性输入操纵自我评估机制。       |   2   | D/V  |
| 10.8.3 | 验证元认知安全分析是否能够识别代理推理过程中的潜在偏见、操纵或妥协。   |   2   | D/V  |
| 10.8.4 | 验证基于反射的安全警告是否触发了增强监控和潜在人为干预的工作流程。    |   3   | D/V  |
| 10.8.5 | 验证从安全反思中持续学习能够提高威胁检测能力，同时不降低合法功能的性能。 |   3   | D/V  |

---

## 10.9 演化与自我改进安全

具备自我修改和进化能力的代理系统的安全控制。

|   #    | Description                      | Level | Role |
| :----: | -------------------------------- | :---: | :--: |
| 10.9.1 | 验证自我修改功能是否仅限于具有正式验证边界的指定安全区域。    |   1   | D/V  |
| 10.9.2 | 确保在实施进化提案之前进行安全影响评估。             |   2   | D/V  |
| 10.9.3 | 验证自我改进机制是否包括具备完整性验证的回滚功能。        |   2   | D/V  |
| 10.9.4 | 验证元学习安全性是否防止了改进算法的对抗性操纵。         |   3   | D/V  |
| 10.9.5 | 验证递归自我改进在形式安全约束下的界限，并提供收敛性的数学证明。 |   3   | D/V  |
