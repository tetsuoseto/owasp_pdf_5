# C1 訓練資料治理與偏差管理

## 控制目標

訓練數據必須以保存來源、保障安全、確保品質和公平性的方式進行取得、處理和維護。這樣做既履行了法律義務，也減少了在人工智慧整個生命週期中出現偏見、中毒或隱私洩露的風險。

---

## C1.1 訓練數據來源

維護所有數據集的可驗證清單，只接受可信來源，並記錄每次變更以便審計。

|   #   | Description                                                         | Level | Role |
| :---: | ------------------------------------------------------------------- | :---: | :--: |
| 1.1.1 | 確認是否維護了一份最新的所有訓練資料來源清單（來源、管理者/擁有者、許可證、收集方法、預期使用限制及處理歷史）。            |   1   | D/V  |
| 1.1.2 | 確保僅允許經過質量、代表性、倫理來源和授權合規性審核的數據集，以降低中毒、內嵌偏見和知識產權侵權的風險。                |   1   | D/V  |
| 1.1.3 | 驗證是否已強制執行資料最小化，以排除不必要或敏感的屬性。                                        |   1   | D/V  |
| 1.1.4 | 確認所有數據集變更均需經過記錄的審批流程。                                               |   2   | D/V  |
| 1.1.5 | 確認標註/註解品質是透過審查者的交叉檢查或共識來確保的。                                        |   2   | D/V  |
| 1.1.6 | 確認是否為重要的訓練資料集維護「資料卡」或「資料集資料表」，詳細說明其特徵、動機、組成、收集過程、預處理，以及建議/不建議的使用方式。 |   2   | D/V  |

---

## C1.2 訓練數據安全與完整性

限制存取權限，對靜態資料與傳輸中的資料進行加密，並驗證完整性以防止竄改或竊取。

|   #   | Description                                                              | Level | Role |
| :---: | ------------------------------------------------------------------------ | :---: | :--: |
| 1.2.1 | 驗證存取控制是否保護存儲和數據管道。                                                       |   1   | D/V  |
| 1.2.2 | 驗證資料集在傳輸過程中已加密，並且對於所有敏感或可識別個人身份資訊（PII）在靜止狀態下也進行加密，使用符合業界標準的加密演算法和金鑰管理做法。 |   2   | D/V  |
| 1.2.3 | 確認在儲存和傳輸過程中使用密碼學雜湊或數位簽章以確保資料完整性，並且應用自動異常檢測技術來防範未授權的修改或損壞，包括針對性的資料中毒攻擊。   |   2   | D/V  |
| 1.2.4 | 驗證數據集版本是否被追蹤以啟用回滾。                                                       |   3   | D/V  |
| 1.2.5 | 確認過時資料已根據資料保留政策、法規要求安全地清除或匿名化，以縮小攻擊面。                                    |   2   | D/V  |

---

## C1.3 表示完整性與公平性

偵測人口統計偏差並應用緩解措施，以確保模型在各群體間的錯誤率公平。

|   #   | Description                                                                             | Level | Role |
| :---: | --------------------------------------------------------------------------------------- | :---: | :--: |
| 1.3.1 | 確認數據集是否已針對法律保護屬性（例如種族、性別、年齡）以及與模型應用領域相關的其他倫理敏感特徵（例如社會經濟地位、地理位置）進行代表性不平衡和潛在偏差的分析。        |   1   | D/V  |
| 1.3.2 | 確認所識別的偏差已通過有文件記錄的策略進行緩解，例如重新平衡、目標數據增強、算法調整（例如預處理、處理中、後處理技術）或重新加權，並評估緩解措施對公平性和整體模型性能的影響。 |   2   | D/V  |
| 1.3.3 | 確認後訓練公平性指標已被評估並記錄。                                                                      |   2   | D/V  |
| 1.3.4 | 確認生命週期偏誤管理政策是否指定了負責人和審查頻率。                                                              |   3   | D/V  |

---

## C1.4 訓練數據標註的質量、完整性與安全性

使用加密保護標籤，並對關鍵類別要求雙重審查。

|   #   | Description                                                           | Level | Role |
| :---: | --------------------------------------------------------------------- | :---: | :--: |
| 1.4.1 | 透過明確的指導方針、審查員交叉核對、共識機制（例如監控標註者間的一致性）以及明確的異議解決流程，確保標註/註解質量。            |   2   | D/V  |
| 1.4.2 | 驗證標籤工件是否應用了密碼雜湊或數位簽章，以確保其完整性和真實性。                                     |   2   | D/V  |
| 1.4.3 | 驗證標註介面和平台能強化存取控制，維持所有標註活動的防篡改審計日誌，並防止未授權的修改。                          |   2   | D/V  |
| 1.4.4 | 確認對安全性、保安性或公平性至關重要的標籤（例如，識別有害內容、關鍵醫療發現）必須進行強制性的獨立雙重審核或同等嚴格的驗證。        |   3   | D/V  |
| 1.4.5 | 確保根據資料最小化原則，標籤中意外捕獲或必須存在的敏感資訊（包括個人識別資訊，PII）在靜態存儲和傳輸過程中已被遮蔽、化名、匿名化或加密。 |   2   | D/V  |
| 1.4.6 | 確認標註指南和說明是否全面、版本受控且經過同儕審查。                                            |   2   | D/V  |
| 1.4.7 | 確認數據架構（包括標籤的架構）已明確定義並受版本控制。                                           |   2   | D/V  |
| 1.8.2 | 確認外包或群眾外包的標記工作流程包含技術和程序上的防護措施，以確保資料的機密性、完整性、標籤品質，並防止資料洩漏。             |   2   | D/V  |

---

## C1.5 訓練數據品質保證

結合自動化驗證、手動抽查及紀錄的修正措施，以保證數據集的可靠性。

|   #   | Description                                                                   | Level | Role |
| :---: | ----------------------------------------------------------------------------- | :---: | :--: |
| 1.5.1 | 驗證自動化測試是否能在每次資料攝取或重大轉換時捕捉格式錯誤、空值及標籤偏差。                                        |   1   |  D   |
| 1.5.2 | 驗證失敗的資料集是否已隔離並具備稽核追蹤記錄。                                                       |   1   | D/V  |
| 1.5.3 | 驗證領域專家手動抽檢涵蓋統計上顯著的樣本量（例如，≥1% 或 1,000 個樣本，以較大者為準，或根據風險評估決定），以識別自動化未能發現的細微質量問題。 |   2   |  V   |
| 1.5.4 | 驗證修復步驟是否已附加到來源記錄中。                                                            |   2   | D/V  |
| 1.5.5 | 確認品質門檻會阻擋不合格的資料集，除非已獲批准例外情況。                                                  |   2   | D/V  |

---

## C1.6 資料中毒檢測

應用統計異常檢測和隔離工作流程，以阻止對抗性插入。

|   #   | Description                                                          | Level | Role |
| :---: | -------------------------------------------------------------------- | :---: | :--: |
| 1.6.1 | 確認在數據輸入及主要訓練執行前，對訓練數據應用異常檢測技術（如統計方法、離群點檢測、嵌入分析）以識別潛在的投毒攻擊或非故意的數據損壞。  |   2   | D/V  |
| 1.6.2 | 確認標記的樣本在訓練前會觸發人工審查。                                                  |   2   | D/V  |
| 1.6.3 | 驗證結果是否反饋至模型的安全檔案，並用以提供持續的威脅情報。                                       |   2   |  V   |
| 1.6.4 | 確認檢測邏輯是否已更新為新的威脅情報。                                                  |   3   | D/V  |
| 1.6.5 | 驗證線上學習管線是否監控分佈漂移。                                                    |   3   | D/V  |
| 1.6.6 | 根據系統的風險概況和數據來源，驗證是否考慮並實施針對已知數據中毒攻擊類型（例如，標籤翻轉、後門觸發插入、影響力實例攻擊）的特定防禦措施。 |   3   | D/V  |

---

## C1.7 使用者資料刪除與同意執行

在所有資料集、備份和派生產物中遵守刪除和撤回同意的請求。

|   #   | Description                                                       | Level | Role |
| :---: | ----------------------------------------------------------------- | :---: | :--: |
| 1.7.1 | 驗證刪除流程是否清除主要和衍生數據，並評估對模型的影響，且評估並在必要時解決受影響模型的影響（例如，透過重新訓練或重新校準）。   |   1   | D/V  |
| 1.7.2 | 確認有機制追蹤並尊重用戶對用於訓練資料的同意範圍及狀態（包括撤回），且在資料納入新的訓練流程或重大模型更新前，會先驗證同意有效性。 |   2   |  D   |
| 1.7.3 | 驗證工作流程是否每年測試並記錄。                                                  |   2   |  V   |

---

## C1.8 供應鏈安全

審查外部數據供應商，並通過已認證且加密的通道驗證數據完整性。

|   #   | Description                                                                   | Level | Role |
| :---: | ----------------------------------------------------------------------------- | :---: | :--: |
| 1.8.1 | 確認第三方資料供應商，包括預訓練模型提供者和外部資料集，在其資料或模型整合之前，已經通過安全性、隱私、倫理採購及資料品質的盡職調查。            |   2   | D/V  |
| 1.8.2 | 驗證外部傳輸是否使用 TLS/身份驗證及完整性檢查。                                                    |   1   |  D   |
| 1.8.3 | 確認高風險資料來源（例如，來源不明的開源數據集、未經審核的供應商）在用於敏感應用前，接受加強審查，如沙盒分析、全面的質量/偏差檢查，以及針對性的污染檢測。 |   2   | D/V  |
| 1.8.4 | 確認從第三方獲取的預訓練模型在微調或部署之前，已針對內嵌偏見、潛在後門、架構完整性以及原始訓練數據的來源進行評估。                     |   3   | D/V  |

---

## C1.9 對抗樣本檢測

在訓練階段實施措施，例如對抗性訓練，以提升模型對對抗性樣本的韌性。

|   #   | Description                                                   | Level | Role |
| :---: | ------------------------------------------------------------- | :---: | :--: |
| 1.9.1 | 根據風險評估，確認對相關模型實施並調整適當的防禦措施，如對抗訓練（使用生成的對抗樣本）、擾動輸入的資料增強或魯棒優化技術。 |   3   | D/V  |
| 1.9.2 | 確認若採用對抗性訓練，對抗性資料集的生成、管理及版本控制皆有文件記錄並受到管控。                      |   2   | D/V  |
| 1.9.3 | 驗證對抗性魯棒性訓練對模型性能（包括對乾淨輸入和對抗性輸入）的影響，以及公平性指標的評估、記錄和監控。           |   3   | D/V  |
| 1.9.4 | 確保針對對抗性訓練與魯棒性的策略定期審查並更新，以應對不斷演變的對抗性攻擊技術。                      |   3   | D/V  |

---

## C1.10 資料血統與可追蹤性

追蹤每個數據點從來源到模型輸入的完整流程，以便進行審計和事件回應。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.10.1 | 確認每個數據點的血統（包括所有轉換、增強和合併）是否已被記錄並且能夠重建。 |   2   | D/V  |
| 1.10.2 | 驗證系譜紀錄是否不可變更、被安全存儲，並可供審計或調查使用。        |   2   | D/V  |
| 1.10.3 | 驗證血統追蹤涵蓋原始資料和合成資料。                    |   2   | D/V  |

---

## C1.11 合成數據管理

確保合成數據得到妥善管理、標記並進行風險評估。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.11.1 | 確認所有合成數據在整個流程中均被明確標示並可與真實數據區分。        |   2   | D/V  |
| 1.11.2 | 確認合成資料的生成過程、參數及預期用途是否已被記錄。            |   2   | D/V  |
| 1.11.3 | 在用於訓練之前，請確認合成資料已進行偏差、隱私洩漏和代表性問題的風險評估。 |   2   | D/V  |

---

## C1.12 資料存取監控與異常檢測

監控並警示對訓練數據的異常存取，以偵測內部威脅或資料外洩。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.12.1 | 驗證所有對訓練數據的訪問均有記錄，包括用戶、時間和操作。          |   2   | D/V  |
| 1.12.2 | 確認是否定期審查存取日誌，以檢測異常模式，例如大量匯出或來自新地點的存取。 |   2   | D/V  |
| 1.12.3 | 確認可疑訪問事件會產生警報並且能夠迅速調查。                |   2   | D/V  |

---

## C1.13 資料保留與過期政策

定義並強制執行資料保留期限，以減少不必要的資料儲存。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| 1.13.1 | 確認所有訓練數據集均已明確定義保存期限。            |   1   | D/V  |
| 1.13.2 | 驗證資料集是否在其生命週期結束時自動過期、刪除或審查是否刪除。 |   2   | D/V  |
| 1.13.3 | 確認保留和刪除操作已被記錄並可審計。              |   2   | D/V  |

---

## C1.14 法規與司法管轄合規性

確保所有訓練數據符合適用的法律和法規。

|   #    | Description                              | Level | Role |
| :----: | ---------------------------------------- | :---: | :--: |
| 1.14.1 | 確認所有資料集均識別並執行資料駐留及跨境傳輸要求。                |   2   | D/V  |
| 1.14.2 | 確認已識別並處理與特定行業相關的法規（例如，醫療保健、金融）在數據處理中的要求。 |   2   | D/V  |
| 1.14.3 | 確認相關隱私法規（例如 GDPR、CCPA）的遵從性已被記錄並定期審查。     |   2   | D/V  |

---

## C1.15 數據水印與指紋識別

檢測未經授權的專有或敏感數據重用或外洩。

|   #    | Description                   | Level | Role |
| :----: | ----------------------------- | :---: | :--: |
| 1.15.1 | 在可行的情況下，驗證數據集或子集是否已加上水印或指紋標記。 |   3   | D/V  |
| 1.15.2 | 驗證浮水印/指紋標記方法不會引入偏見或隱私風險。      |   3   | D/V  |
| 1.15.3 | 確認定期進行檢查，以偵測水印資料的未經授權重複使用或外洩。 |   3   | D/V  |

---

## C1.16 資料主體權利管理

支持資料主體權利，如存取、更正、限制和反對。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| 1.16.1 | 驗證是否存在回應資料主體關於存取、修正、限制或反對請求的機制。 |   2   | D/V  |
| 1.16.2 | 驗證請求是否在法定時限內被記錄、追蹤並完成。          |   2   | D/V  |
| 1.16.3 | 驗證資料主體權利流程是否定期進行測試與審查以確保其有效性。   |   2   | D/V  |

---

## C1.17 資料集版本影響分析

在更新或替換資料集版本之前，評估資料集變更的影響。

|   #    | Description                             | Level | Role |
| :----: | --------------------------------------- | :---: | :--: |
| 1.17.1 | 在更新或替換資料集版本之前，確認已執行影響分析，涵蓋模型效能、公平性及合規性。 |   2   | D/V  |
| 1.17.2 | 確認影響分析結果已記錄並由相關利益相關者審查。                 |   2   | D/V  |
| 1.17.3 | 確認在新版本引入不可接受的風險或回歸時存在回滾計劃。              |   2   | D/V  |

---

## C1.18 數據標註人力安全

確保參與數據註釋人員的安全與完整性。

|   #    | Description                             | Level | Role |
| :----: | --------------------------------------- | :---: | :--: |
| 1.18.1 | 確認所有參與數據標註的人員均已通過背景審查，並接受過數據安全與隱私保護的培訓。 |   2   | D/V  |
| 1.18.2 | 確認所有註解人員簽署保密與不揭露協議。                     |   2   | D/V  |
| 1.18.3 | 確認標註平台執行存取控制並監控內部威脅。                    |   2   | D/V  |
