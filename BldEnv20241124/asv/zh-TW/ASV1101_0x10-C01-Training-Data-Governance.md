# C1 訓練數據治理與偏差管理

## 控制目標

訓練資料必須以能保留來源、保障安全、確保品質及公平性的方式取得、處理與維護。如此可履行法律責任，並降低在整個 AI 生命週期中出現偏見、中毒或隱私洩露的風險。

---

## C1.1 訓練數據來源

維護所有資料集的可驗證清單，只接受可信來源，並記錄每次更改以進行審計。

|   #   | Description                                                         | Level | Role |
| :---: | ------------------------------------------------------------------- | :---: | :--: |
| 1.1.1 | 確認已維護每個訓練資料來源的最新清單（來源、管理者/擁有者、授權、收集方法、預期使用限制及處理歷史）。                 |   1   | D/V  |
| 1.1.2 | 確保僅允許經過品質、代表性、倫理採購及授權合規性審核的資料集，降低中毒風險、內嵌偏見及智慧財產權侵害的風險。              |   1   | D/V  |
| 1.1.3 | 驗證是否強制執行資料最小化，以排除不必要或敏感的屬性。                                         |   1   | D/V  |
| 1.1.4 | 確認所有數據集的更改均需經過紀錄的批准工作流程。                                            |   2   | D/V  |
| 1.1.5 | 確保標籤/註解質量通過審核者交叉檢查或共識來保證。                                           |   2   | D/V  |
| 1.1.6 | 確認是否為重要的訓練資料集維護「資料卡」或「資料集說明書」，詳細說明其特徵、動機、組成、收集過程、前處理，以及建議/不建議的使用方式。 |   2   | D/V  |

---

## C1.2 訓練數據安全性與完整性

限制存取，對靜態和傳輸中的資料進行加密，並驗證完整性以防止篡改或竊取。

|   #   | Description                                                                    | Level | Role |
| :---: | ------------------------------------------------------------------------------ | :---: | :--: |
| 1.2.1 | 驗證存取控制是否保護儲存和資料管線。                                                             |   1   | D/V  |
| 1.2.2 | 驗證資料集在傳輸過程中已加密，並針對所有敏感或個人身份識別資訊（PII）在靜態狀態下也使用業界標準的加密演算法及金鑰管理實務進行保護。            |   2   | D/V  |
| 1.2.3 | 驗證在資料儲存與傳輸過程中，是否使用加密雜湊或數位簽章以確保資料完整性，並且是否應用了自動異常檢測技術，以防範未授權的修改或損壞，包括針對性的資料污染攻擊。 |   2   | D/V  |
| 1.2.4 | 確認資料集版本已被追蹤以支持回滾。                                                              |   3   | D/V  |
| 1.2.5 | 驗證過時資料已依據資料保留政策、法規要求安全地清除或匿名化，以縮小攻擊面。                                          |   2   | D/V  |

---

## C1.3 表示完整性與公平性

偵測人口統計偏差並採取緩解措施，使模型的錯誤率在各群體間公平。

|   #   | Description                                                                                | Level | Role |
| :---: | ------------------------------------------------------------------------------------------ | :---: | :--: |
| 1.3.1 | 確認數據集是否已針對具代表性不平衡及可能存在偏見的情況進行分析，涵蓋法律保護屬性（如種族、性別、年齡）以及與模型應用領域相關的其他倫理敏感特徵（如社會經濟狀況、地理位置）。     |   1   | D/V  |
| 1.3.2 | 確認已識別的偏差透過有文件記錄的策略進行緩解，例如重新平衡、有針對性的數據增強、演算法調整（例如前處理、中處理、後處理技術）或重新加權，並評估緩解措施對公平性及整體模型效能的影響。 |   2   | D/V  |
| 1.3.3 | 確認已評估並記錄訓練後的公平性指標。                                                                         |   2   | D/V  |
| 1.3.4 | 驗證生命週期偏差管理政策是否指定了負責人和審查頻率。                                                                 |   3   | D/V  |

---

## C1.4 訓練數據標籤的品質、完整性與安全性

對標籤進行加密保護，並對關鍵類別要求雙重審核。

|   #   | Description                                                                  | Level | Role |
| :---: | ---------------------------------------------------------------------------- | :---: | :--: |
| 1.4.1 | 確保標註/註解品質通過明確的指導方針、審核員交叉核對、共識機制（例如，監控標註者間一致性）以及明確的異議解決流程。                    |   2   | D/V  |
| 1.4.2 | 驗證密碼學雜湊或數位簽章是否應用於標籤工件，以確保其完整性和真實性。                                           |   2   | D/V  |
| 1.4.3 | 驗證標註介面和平台是否強制執行強健的存取控制，維護所有標註活動的防篡改審計日誌，並防止未經授權的修改。                          |   2   | D/V  |
| 1.4.4 | 確保對安全性、保安性或公平性至關重要的標籤（例如，識別有毒內容、關鍵醫療發現）必須進行強制性獨立雙重審核或同等的嚴格驗證。                |   3   | D/V  |
| 1.4.5 | 確認根據資料最小化原則，敏感資訊（包括個人可識別資訊，PII）若在標籤中無意中被捕捉或必然存在，已在靜態和傳輸過程中進行遮蔽、假名化、匿名化或加密處理。 |   2   | D/V  |
| 1.4.6 | 確認標註指南和說明是全面的、版本可控的，並且經過同儕審查。                                                |   2   | D/V  |
| 1.4.7 | 確認資料結構（包括標籤）已明確定義並進行版本控制。                                                    |   2   | D/V  |
| 1.8.2 | 確認外包或群眾外包標註工作流程包含技術/程序上的保障措施，以確保資料的機密性、完整性、標註品質，並防止資料外洩。                     |   2   | D/V  |

---

## C1.5 訓練資料品質保證

結合自動化驗證、手動抽查及記錄的修正措施，以確保數據集的可靠性。

|   #   | Description                                                                         | Level | Role |
| :---: | ----------------------------------------------------------------------------------- | :---: | :--: |
| 1.5.1 | 驗證自動化測試是否在每次資料攝取或重大轉換時捕捉格式錯誤、空值和標籤偏移。                                               |   1   |  D   |
| 1.5.2 | 驗證失敗的資料集是否被隔離並附有審計追蹤。                                                               |   1   | D/V  |
| 1.5.3 | 驗證領域專家的手動抽檢是否涵蓋具有統計顯著性的樣本（例如，≥1% 或 1,000 個樣本，以較大者為準，或依風險評估結果決定），以識別自動化未能檢測出的細微品質問題。 |   2   |  V   |
| 1.5.4 | 驗證修復步驟是否已附加到來源記錄中。                                                                  |   2   | D/V  |
| 1.5.5 | 確認品質門檻會阻擋不合格的資料集，除非已批准例外情況。                                                         |   2   | D/V  |

---

## C1.6 資料中毒偵測

應用統計異常檢測和隔離工作流程來阻止對抗性插入。

|   #   | Description                                                              | Level | Role |
| :---: | ------------------------------------------------------------------------ | :---: | :--: |
| 1.6.1 | 確認異常檢測技術（例如，統計方法、異常值檢測、嵌入分析）已應用於資料擷取階段的訓練資料及主要訓練執行前，以識別潛在的污染攻擊或非故意的資料損壞。 |   2   | D/V  |
| 1.6.2 | 確認被標記的樣本在訓練前會觸發人工審查。                                                     |   2   | D/V  |
| 1.6.3 | 驗證結果是否用於餵養模型的安全檔案，並通報持續的威脅情報。                                            |   2   |  V   |
| 1.6.4 | 確認檢測邏輯已隨新的威脅情報更新。                                                        |   3   | D/V  |
| 1.6.5 | 驗證線上學習管線是否監控分佈漂移。                                                        |   3   | D/V  |
| 1.6.6 | 確認是否根據系統的風險評估和資料來源，考慮並實施針對已知資料中毒攻擊類型（例如標籤翻轉、後門觸發插入、影響力實例攻擊）的特定防禦措施。      |   3   | D/V  |

---

## C1.7 用戶數據刪除與同意執行

尊重並執行跨資料集、備份及衍生產物的刪除和撤回同意請求。

|   #   | Description                                                           | Level | Role |
| :---: | --------------------------------------------------------------------- | :---: | :--: |
| 1.7.1 | 確認刪除工作流程會清除主要和衍生資料，並評估對模型的影響，且對受影響的模型進行評估，如有必要，採取相應措施（例如透過重新訓練或重新校準）。 |   1   | D/V  |
| 1.7.2 | 確認已建立機制以追蹤並尊重用戶對訓練數據的同意範圍及狀態（包括撤回），並且在將數據納入新的訓練流程或重大模型更新之前，驗證同意的有效性。  |   2   |  D   |
| 1.7.3 | 確認工作流程已經每年測試並記錄。                                                      |   2   |  V   |

---

## C1.8 供應鏈安全

審核外部數據提供者並通過認證的加密通道驗證完整性。

|   #   | Description                                                                              | Level | Role |
| :---: | ---------------------------------------------------------------------------------------- | :---: | :--: |
| 1.8.1 | 確認第三方資料供應商，包括預先訓練模型和外部資料集的提供者，在其資料或模型被整合之前，已經進行安全性、隱私、倫理採購及資料品質的盡職調查。                    |   2   | D/V  |
| 1.8.2 | 確認外部傳輸是否使用 TLS/身份驗證和完整性檢查。                                                               |   1   |  D   |
| 1.8.3 | 確保對高風險數據來源（例如，來源不明的開源數據集、未經審核的供應商）進行加強審查，例如沙盒分析、廣泛的質量/偏差檢查以及針對性的數據中毒檢測，並在用於敏感應用之前完成這些步驟。 |   2   | D/V  |
| 1.8.4 | 確認從第三方獲取的預訓練模型在微調或部署之前，已對其嵌入的偏見、潛在的後門、架構完整性及其原始訓練數據的來源進行評估。                              |   3   | D/V  |

---

## C1.9 對抗樣本檢測

在訓練階段中實施措施，例如對抗式訓練，以增強模型對對抗樣本的韌性。

|   #   | Description                                                       | Level | Role |
| :---: | ----------------------------------------------------------------- | :---: | :--: |
| 1.9.1 | 確認根據風險評估，已為相關模型實施並調整適當的防禦措施，例如對抗訓練（使用生成的對抗樣本）、帶有擾動輸入的資料增強或魯棒優化技術。 |   3   | D/V  |
| 1.9.2 | 確認如果使用對抗訓練，則對抗資料集的生成、管理和版本控制都有被記錄和管控。                             |   2   | D/V  |
| 1.9.3 | 確認已評估、記錄並監控對抗性魯棒性訓練對模型效能（包括對乾淨輸入和對抗性輸入）及公平性指標的影響。                 |   3   | D/V  |
| 1.9.4 | 確認對抗性訓練和韌性策略定期進行審查和更新，以對抗不斷演變的對抗性攻擊技術。                            |   3   | D/V  |

---

## C1.10 數據血緣與可追溯性

追蹤每個數據點從來源到模型輸入的完整過程，以利審計性和事件回應。

|   #    | Description                       | Level | Role |
| :----: | --------------------------------- | :---: | :--: |
| 1.10.1 | 確認每個數據點的來源，包括所有轉換、增強和合併，皆有記錄並能重建。 |   2   | D/V  |
| 1.10.2 | 驗證血統記錄是否不可更改、安全存儲，且可供審計或調查使用。     |   2   | D/V  |
| 1.10.3 | 驗證血統追蹤涵蓋原始數據和合成數據。                |   2   | D/V  |

---

## C1.11 合成數據管理

確保合成數據得到妥善管理、標記並進行風險評估。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| 1.11.1 | 確認所有合成資料在整個流程中均有清楚標示並與真實資料明顯區分。      |   2   | D/V  |
| 1.11.2 | 確認合成數據的生成過程、參數及預期用途已被記錄。             |   2   | D/V  |
| 1.11.3 | 在用於訓練前，請確保對合成數據進行偏差、隱私洩漏及表示性問題的風險評估。 |   2   | D/V  |

---

## C1.12 資料存取監控與異常偵測

監控並警示對訓練數據的異常存取，以偵測內部威脅或資料外洩。

|   #    | Description                           | Level | Role |
| :----: | ------------------------------------- | :---: | :--: |
| 1.12.1 | 確認所有對訓練數據的存取皆有記錄，包括使用者、時間與操作。         |   2   | D/V  |
| 1.12.2 | 確認存取日誌是否定期檢查，以辨識異常模式，例如大量匯出或來自新地點的存取。 |   2   | D/V  |
| 1.12.3 | 確認可疑存取事件是否會產生警示並被立即調查。                |   2   | D/V  |

---

## C1.13 數據保留與過期政策

定義並執行資料保留期限，以最小化不必要的資料儲存。

|   #    | Description                    | Level | Role |
| :----: | ------------------------------ | :---: | :--: |
| 1.13.1 | 確認所有訓練數據集皆明確定義保留期限。            |   1   | D/V  |
| 1.13.2 | 確認資料集在其生命週期結束時能自動過期、刪除或進行刪除審查。 |   2   | D/V  |
| 1.13.3 | 驗證保留和刪除操作是否已記錄並可稽核。            |   2   | D/V  |

---

## C1.14 法規與司法管轄遵循

確保所有訓練數據均符合適用的法律和法規。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| 1.14.1 | 確認所有數據集的數據駐留和跨境傳輸要求已被識別並強制執行。        |   2   | D/V  |
| 1.14.2 | 確認已識別並處理特定行業規範（例如，醫療保健、金融）在數據處理中的要求。 |   2   | D/V  |
| 1.14.3 | 確認相關隱私法規（例如 GDPR、CCPA）的合規性已被記錄並定期審查。 |   2   | D/V  |

---

## C1.15 資料水印與指紋辨識

偵測未經授權的專有或敏感資料重用或洩漏。

|   #    | Description                  | Level | Role |
| :----: | ---------------------------- | :---: | :--: |
| 1.15.1 | 在可行的情況下，確認資料集或其子集已加水印或指紋識別。  |   3   | D/V  |
| 1.15.2 | 驗證水印/指紋技術不會引入偏差或隱私風險。        |   3   | D/V  |
| 1.15.3 | 確認定期進行檢查以偵測未經授權的重複使用或水印資料外洩。 |   3   | D/V  |

---

## C1.16 資料主體權利管理

支持資料主體權利，如存取、更正、限制和異議。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| 1.16.1 | 確認已建立機制以回應資料主體提出的存取、更正、限制或反對請求。 |   2   | D/V  |
| 1.16.2 | 確認請求在法律規定的時間範圍內被記錄、追蹤並完成。       |   2   | D/V  |
| 1.16.3 | 驗證資料主體權利流程是否定期進行測試和審查以確保其有效性。   |   2   | D/V  |

---

## C1.17 資料集版本影響分析

在更新或替換版本之前，評估數據集變更的影響。

|   #    | Description                            | Level | Role |
| :----: | -------------------------------------- | :---: | :--: |
| 1.17.1 | 確認在更新或替換資料集版本前，有執行影響分析，涵蓋模型效能、公平性及合規性。 |   2   | D/V  |
| 1.17.2 | 確認影響分析結果已被相關利害關係人記錄並審查。                |   2   | D/V  |
| 1.17.3 | 確認存在回滾計劃，以防新版本引入不可接受的風險或回退情況。          |   2   | D/V  |

---

## C1.18 資料標註工作人員安全

確保參與數據註釋人員的安全與完整性。

|   #    | Description                         | Level | Role |
| :----: | ----------------------------------- | :---: | :--: |
| 1.18.1 | 確認所有參與資料標註的人員均已通過背景調查並接受資料安全與隱私的培訓。 |   2   | D/V  |
| 1.18.2 | 確認所有註解人員簽署保密及不外洩協議。                 |   2   | D/V  |
| 1.18.3 | 驗證標註平台是否實施訪問控制並監控內部威脅。              |   2   | D/V  |
