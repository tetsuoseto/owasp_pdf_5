# 附錄 C：人工智慧安全治理與文件管理

## 目標

本附錄提供建立組織架構、政策及流程的基本要求，以管理整個系統生命週期中的人工智慧安全。

---

## AC.1 人工智慧風險管理框架採用

提供一個正式框架，以識別、評估並減輕整個系統生命週期中的人工智慧特定風險。

|   #    | Description                          | Level | Role |
| :----: | ------------------------------------ | :---: | :--: |
| AC.1.1 | 確認已制定並實施針對人工智慧的風險評估方法。               |   1   | D/V  |
| AC.1.2 | 確認在人工智慧生命週期的關鍵階段及重大變更之前進行風險評估。       |   2   |  D   |
| AC.1.3 | 驗證風險管理框架是否與既定標準（例如 NIST AI RMF）保持一致。 |   3   | D/V  |

---

## AC.2 人工智慧安全政策與程序

定義並強制執行組織標準，以確保 AI 的安全開發、部署與運營。

|   #    | Description                      | Level | Role |
| :----: | -------------------------------- | :---: | :--: |
| AC.2.1 | 確認已制定人工智慧安全政策並有文件記錄。             |   1   | D/V  |
| AC.2.2 | 確認政策至少每年審查和更新一次，並在重要威脅環境變化後進行更新。 |   2   |  D   |
| AC.2.3 | 確認政策涵蓋所有 AISVS 類別及適用的法規要求。       |   3   | D/V  |

---

## AC.3 人工智慧安全的角色與職責

在整個組織中建立明確的人工智慧安全責任制度。

|   #    | Description                        | Level | Role |
| :----: | ---------------------------------- | :---: | :--: |
| AC.3.1 | 確認已記錄人工智慧安全的角色與職責。                 |   1   | D/V  |
| AC.3.2 | 確認相關負責人員具備適當的安全專業知識。               |   2   |  D   |
| AC.3.3 | 確認是否已為高風險人工智慧系統建立了人工智慧倫理委員會或治理委員會。 |   3   | D/V  |

---

## AC.4 倫理人工智慧指導原則執行

確保人工智慧系統依照既定的倫理原則運作。

|   #    | Description             | Level | Role |
| :----: | ----------------------- | :---: | :--: |
| AC.4.1 | 確認存在人工智慧開發與部署的倫理指導方針。   |   1   | D/V  |
| AC.4.2 | 確認已建立機制以偵測及報告倫理違規行為。    |   2   |  D   |
| AC.4.3 | 確認已對已部署的人工智慧系統定期進行倫理審查。 |   3   | D/V  |

---

## AC.5 人工智慧法規遵循監控

保持對不斷變化的人工智慧法規的意識並遵守相關規定。

|   #    | Description               | Level | Role |
| :----: | ------------------------- | :---: | :--: |
| AC.5.1 | 確認存在識別適用之人工智慧法規的流程。       |   1   | D/V  |
| AC.5.2 | 確保已評估符合所有法規要求。            |   2   |  D   |
| AC.5.3 | 確保監管變更能觸發對人工智慧系統的及時審查和更新。 |   3   | D/V  |
