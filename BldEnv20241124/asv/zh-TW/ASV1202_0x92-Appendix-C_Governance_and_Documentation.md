# 附錄C：人工智慧安全治理與文件管理

## 目標

本附錄提供建立組織結構、政策和流程的基本要求，以管理整個系統生命週期中的人工智慧安全。

---

## AC.1 AI 風險管理框架採用

提供一個正式框架，以識別、評估和減輕整個系統生命週期中的人工智慧特有風險。

|   #    | Description                       | Level | Role |
| :----: | --------------------------------- | :---: | :--: |
| AC.1.1 | 確認已記錄並實施專門針對人工智慧的風險評估方法。          |   1   | D/V  |
| AC.1.2 | 確認在人工智慧生命週期的關鍵點以及重大變更前進行風險評估。     |   2   |  D   |
| AC.1.3 | 確認風險管理框架是否符合既定標準（例如，NIST AI RMF）。 |   3   | D/V  |

---

## AC.2 人工智慧安全政策與程序

定義並執行組織標準，以確保人工智慧的安全開發、部署及運營。

|   #    | Description                            | Level | Role |
| :----: | -------------------------------------- | :---: | :--: |
| AC.2.1 | 確認已存在已記錄的人工智慧安全政策。                     |   1   | D/V  |
| AC.2.2 | 確保政策至少每年審查和更新一次，並在重大威脅態勢變化後進行更新。       |   2   |  D   |
| AC.2.3 | 確認政策涵蓋所有人工智慧系統驗證與安全性（AISVS）類別及適用的法規要求。 |   3   | D/V  |

---

## AC.3 人工智慧安全的角色與職責

在整個組織中建立明確的 AI 安全責任制度。

|   #    | Description                     | Level | Role |
| :----: | ------------------------------- | :---: | :--: |
| AC.3.1 | 確認 AI 安全角色和責任已被記錄。              |   1   | D/V  |
| AC.3.2 | 確認負責人員具備適當的安全專業知識。              |   2   |  D   |
| AC.3.3 | 確認已為高風險人工智慧系統設立人工智慧倫理委員會或治理委員會。 |   3   | D/V  |

---

## AC.4 道德人工智慧指導原則的執行

確保人工智慧系統依據既定的倫理原則運作。

|   #    | Description            | Level | Role |
| :----: | ---------------------- | :---: | :--: |
| AC.4.1 | 確認人工智慧開發與部署的倫理準則存在。    |   1   | D/V  |
| AC.4.2 | 確認已建立機制以偵測並報告倫理違規行為。   |   2   |  D   |
| AC.4.3 | 確認已對部署的 AI 系統定期進行倫理審查。 |   3   | D/V  |

---

## AC.5 人工智慧法規遵循監控

保持對不斷發展的人工智慧法規的認識與遵守。

|   #    | Description               | Level | Role |
| :----: | ------------------------- | :---: | :--: |
| AC.5.1 | 驗證是否存在識別適用人工智慧法規的流程。      |   1   | D/V  |
| AC.5.2 | 確認已評估符合所有法規要求。            |   2   |  D   |
| AC.5.3 | 確認監管變更能觸發對人工智慧系統的及時審查和更新。 |   3   | D/V  |
